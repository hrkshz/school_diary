### 3.4 テストデータ作成スクリプト

```python
# school_diary/diary/management/commands/setup_test_data.py

from django.core.management.base import BaseCommand
from django.contrib.auth.models import User
from diary.models import ClassRoom, DiaryEntry, TeacherNote
from datetime import date, timedelta

class Command(BaseCommand):
    help = 'テストデータを作成'

    def handle(self, *args, **options):
        # 管理者作成
        admin, _ = User.objects.get_or_create(
            username='admin',
            defaults={
                'email': 'admin@example.com',
                'is_staff': True,
                'is_superuser': True,
            }
        )
        admin.set_password('password123')
        admin.save()

        # 担任作成
        teacher1a, _ = User.objects.get_or_create(
            username='teacher1a',
            defaults={
                'email': 'teacher1a@example.com',
                'first_name': '太郎',
                'last_name': '先生',
                'is_staff': True,
            }
        )
        teacher1a.set_password('password123')
        teacher1a.save()

        # クラス作成
        class1a, _ = ClassRoom.objects.get_or_create(
            grade=1,
            class_name='A',
            academic_year=2025,
            defaults={'homeroom_teacher': teacher1a}
        )

        # 生徒作成
        for i in range(1, 6):
            student, _ = User.objects.get_or_create(
                username=f'student1a{i:02d}',
                defaults={
                    'email': f'student1a{i:02d}@example.com',
                    'first_name': f'生徒{i}',
                    'last_name': '山田',
                }
            )
            student.set_password('password123')
            student.save()
            class1a.students.add(student)

        # 連絡帳エントリー作成
        student1 = User.objects.get(username='student1a01')
        for days_ago in [0, 1, 2]:
            entry_date = date.today() - timedelta(days=days_ago)
            entry, _ = DiaryEntry.objects.get_or_create(
                student=student1,
                entry_date=entry_date,
                defaults={
                    'health_condition': 4,
                    'mental_condition': 5,
                    'reflection': f'{entry_date}の振り返り内容',
                    'is_read': True,
                    'read_by': teacher1a,
                }
            )

        self.stdout.write(self.style.SUCCESS('テストデータ作成完了'))
```

## 4. テストシナリオ

### 4.1 基本機能テスト（課題1）

#### シナリオ1: 生徒が連絡帳を提出

**目的**: 生徒が連絡帳を作成・提出できることを確認

**手順**:
1. student1a03でログイン
   ```
   URL: /login
   Username: student1a03
   Password: password123
   ```
2. ダッシュボードから「連絡帳を書く」をクリック
3. 以下を入力:
   - 体調: 4（良い）
   - メンタル: 5（とても元気）
   - 振り返り: 「今日は数学のテストがあった。頑張った。」
4. 「提出」ボタンをクリック

**期待結果**:
- ✅ 成功メッセージ「連絡帳を提出しました」が表示される
- ✅ ダッシュボードに戻る
- ✅ 「本日の連絡帳: 提出済み ✓」と表示される

---

#### シナリオ2: 担任が既読処理

**目的**: 担任が生徒の連絡帳を確認し、既読処理できることを確認

**手順**:
1. teacher1aでログイン
2. ダッシュボードから「提出状況を見る」をクリック
3. 日付を「2025-10-07（本日）」に設定
4. student1a02（未読）の行をクリック
5. 連絡帳詳細を確認
6. 「イイネ（既読）」ボタンをクリック

**期待結果**:
- ✅ 成功メッセージ「既読処理が完了しました」が表示される
- ✅ 提出状況一覧に戻る
- ✅ student1a02の既読欄に「✓」が表示される

---

#### シナリオ3: 生徒が過去記録を閲覧

**目的**: 生徒が自分の過去記録を閲覧できることを確認

**手順**:
1. student1a04でログイン

3. 過去記録一覧が表示される
4. 特定の日付（例: 2025-10-06）をクリック

**期待結果**:
- ✅ 過去記録一覧が日付順（新しい順）で表示される
- ✅ 詳細画面で内容が表示される
- ✅ 既読マーク「✓」が表示される

#### シナリオ4: 担任が提出状況を確認

**目的**: 担任が1日の提出状況を確認できることを確認

**手順**:
1. teacher1aでログイン
2. ダッシュボードから「提出状況を見る」をクリック

**期待結果**:
- ✅ 1年A組の生徒5名全員が表示される

## 5. 検証観点

### 5.1 機能面

| 観点 | 確認項目 |
|-----|---------|
| **基本機能** | 全ての必須機能が動作する |
| **エラーハンドリング** | 適切なエラーメッセージが表示される |
| **データ整合性** | 制約（unique_together等）が機能する |
| **権限制御** | ロール別に適切な機能が提供される |

### 5.2 ユーザビリティ

| 観点 | 確認項目 |
|-----|---------|
| **画面遷移** | 直感的な画面遷移ができる |
| **操作性** | ボタン・リンクが適切に配置されている |
| **フィードバック** | 操作結果が明確にわかる（成功・エラー） |
| **一貫性** | 同じ操作は同じ結果になる |

### 5.3 セキュリティ

| 観点 | 確認項目 |
|-----|---------|
| **認証** | ログインしないとアクセスできない |
| **認可** | 他人のデータにアクセスできない |
| **CSRF対策** | フォーム送信時にCSRFトークンが機能する |
| **編集制約** | 既読後は誰も編集できない |

### 5.4 パフォーマンス

| 観点 | 確認項目 |
|-----|---------|
| **レスポンス時間** | 通常操作は2秒以内 |
| **データベースクエリ** | N+1問題が発生しない |
| **ページネーション** | 大量データでも表示が遅延しない |

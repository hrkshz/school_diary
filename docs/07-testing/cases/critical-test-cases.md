### TC-AUTH-001-01: 正しい認証情報でログイン成功（正常系）

**Feature**: ユーザー認証
  ユーザーとして、システムにログインしたい

**Scenario**: 正しい認証情報でログイン成功

**Given** (前提条件):
- ユーザーアカウントが存在する（student_1_a_01@example.com / password123）
- ログアウト状態

**When** (操作):
1. ブラウザで https://d2wk3j2pacp33b.cloudfront.net にアクセス
2. ログインページにリダイレクトされる
3. メールアドレス「student_1_a_01@example.com」を入力
4. パスワード「password123」を入力
5. 「ログイン」ボタンをクリック

**Then** (期待結果):
- ログイン成功
- 生徒ダッシュボード（/diary/student/dashboard/）にリダイレクトされる
- ページに「連絡帳を書く」ボタンが表示される

## TC-STU-002: 連絡帳作成

### TC-STU-002-01: 連絡帳新規作成成功（正常系）

**Feature**: 連絡帳作成機能
  生徒として、前日の振り返りを提出したい

**Scenario**: 生徒が連絡帳を新規作成

**Given** (前提条件):
- 生徒アカウント（student_1_a_01@example.com）でログイン済み

**When** (操作):
1. 生徒ダッシュボードで「連絡帳を書く」ボタンをクリック
2. 連絡帳作成画面が表示される

**Then** (期待結果):
- 「連絡帳を提出しました」メッセージが表示される
- 生徒ダッシュボードにリダイレクトされる
- 提出した連絡帳が一覧に表示される
- 担任ダッシュボードの「未読」カテゴリに表示される

  担任として、優先度順に生徒を確認したい

**Given** (前提条件):
- 担任アカウント（teacher_1_a@example.com）でログイン済み

- 連絡帳が複数提出されている（P0、P1、未提出、未読など）

1. 担任ダッシュボード（/diary/teacher/dashboard/）にアクセス

- 6カテゴリが表示される:
  - P0: 重要（🔴マーク、3日連続低下またはメンタル★1）
  - P1: 要注意（🟡マーク）
  - 未提出（昨日未提出の生徒）
  - 未読（既読処理が必要）
  - 反応待ち（既読済みだが反応未追加）

- 各カテゴリに適切な生徒がリストされる
- 生徒カードに3日履歴が表示される

### TC-TEA-ACT-001-01: 既読処理成功、反応追加（正常系）

**Feature**: 既読処理機能
  担任として、連絡帳を確認したことを生徒に伝えたい

**Scenario**: 既読処理と同時に反応を追加

**Given** (前提条件):

- 「既読処理しました」メッセージが表示される
- 生徒Aのカードが「未読」から「完了済み」カテゴリに移動
- 生徒Aの連絡帳詳細に「👍 いいね」が表示される
- 生徒ダッシュボードに「👍 いいね」が表示される

# ITIL プロセス整理

ステータス: 後段
優先度: 低

---

## なぜやるか

- 実装した仕組みを ITIL の文脈で整理し、ドキュメントとして残す
- 面接で「ITIL の考え方をどう適用したか」を説明できるようにする

## 何をするか

- Incident Management: 重大度ルール、起票ルール
- Event Management: イベント分類、正規化ルール
- Change Enablement: 変更手順、承認フロー
- Problem Management: 根本原因分析の考え方
- Service Continuity: 復旧手順、RTO/RPO

## コスト

無料（ドキュメント作成のみ）

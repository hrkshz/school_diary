# SLI/SLO の定義とエラーバジェット

ステータス: 予定
優先度: 中

---

## なぜやるか

- 「サービスが正常か」を定量的に判断する基準がない
- SLI（Service Level Indicator）で「何を測るか」を定義
- SLO（Service Level Objective）で「どのレベルを目指すか」を定義
- エラーバジェットで「どこまで壊れても許容範囲か」を明確にする
- SRE の基本概念を実践で理解する

## 何をするか

- SLI の定義: 可用性（成功リクエスト率）、レイテンシ（p99 応答時間）
- SLO の設定: 例）可用性 99.5%、p99 レイテンシ 3 秒以内
- CloudWatch メトリクスから SLI を算出
- エラーバジェットの消費状況をダッシュボードに表示

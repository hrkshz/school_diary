# インシデント対応の振り返り（Postmortem）

ステータス: 後段
優先度: 低

---

## なぜやるか

- 障害が起きたとき「何が起きたか」「なぜ起きたか」「再発防止は何か」を記録する文化を作る
- Google SRE の Postmortem 文化を取り入れる
- Blame-free（個人を責めない）な振り返りプロセス

## 何をするか

- Postmortem テンプレート（Markdown）を作成
- 項目: タイムライン、影響範囲、根本原因、対応内容、再発防止策
- 過去のインシデント（2025/10/25 メモリ枯渇）を Postmortem として記録

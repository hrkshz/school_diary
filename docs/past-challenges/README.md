## 💡 LabAppでの活用方法

このプロジェクト（school_diary）は、これらの過去課題から抽出された**共通パターン**を実装した基盤です。

### kitsとの対応関係

| 課題の要求 | school_diaryのkits | 説明 |
|-----------|-------------|------|
| ユーザー・ロール管理 | `kits.accounts` | テストユーザー作成、グループ管理 |
| 承認フロー | `kits.approvals` | FSMによる状態管理 |
| 変更履歴 | `kits.audit` | django-simple-history |
| データ可視化 | `kits.reports` | グラフ生成、レポート |
| 通知機能 | `kits.notifications` | メール・プッシュ通知 |
| 検索機能 | `kits.search` | 全文検索 |
| データ入出力 | `kits.io` | CSV/Excelインポート |

### デモアプリケーション

`kits.demos`の`DemoRequest`モデルは、これらの課題で頻出する「申請・承認フロー」のリファレンス実装です。

```python
# DemoRequestの状態遷移は全課題に共通
draft → submitted → approved/denied
```

## 🚀 過去課題を再実装する方法

### ステップ1: kitsの活用
既存のkitsを組み合わせて基本機能を構築

```bash
# 例: 残業管理システムの場合
# 1. モデル定義（DemoRequestを参考に）
# 2. kits.accountsでロール設定
# 3. kits.approvalsで承認フロー
# 4. kits.auditで履歴管理
```

### ステップ2: ドメイン固有のカスタマイズ
各課題特有の機能を追加実装

```python
# 例: 野球部の測定項目
class MeasurementRecord(models.Model):
    sprint_50m = models.DecimalField(...)
    bench_press = models.DecimalField(...)
    # 測定項目に応じたフィールド定義
```

Bootstrap 5を使った管理画面のカスタマイズ

## 📖 学習のポイント

### 課題1から学ぶこと
- **レガシーシステム移行**: 既存運用との整合性
- **ビジネスルール**: 段階的な変更への配慮
- **エンドユーザー視点**: 現場の受け入れやすさ

### 課題2から学ぶこと
- **マルチデバイス対応**: レスポンシブデザイン
- **プライバシー**: 個人情報保護法への準拠
- **データ可視化**: 成長曲線、グラフ表示

### 課題3から学ぶこと
- **データ分析**: 統計処理、トレンド分析
- **チーム管理**: 多人数のデータ管理
- **意思決定支援**: ダッシュボード設計

### 課題4から学ぶこと
- **システム統合**: 複数の既存システムの統合・データ移行
- **横断検索**: 分散データの統合検索
- **予約・在庫管理**: 順番待ち、在庫状態管理
- **定期処理**: 週次バッチ処理、自動通知

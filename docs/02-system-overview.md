# システム概要書

> **対象**: 連絡帳管理システム（school_diary）
> **バージョン**: v0.3.0-map
> **作成日**: 2025-10-22

---

## プロジェクト概要

### 目的
中学校における生徒と担任間の日常的なコミュニケーションをデジタル化し、生徒のメンタルヘルスや体調の変化を早期に検知する。

### 対象ユーザー
- 生徒、担任、学年主任、校長/教頭、システム管理者（5ロール）

---

## 主要機能

### 生徒
- 連絡帳の作成・編集・削除
- 体調・メンタル状態の5段階評価
- 過去の連絡帳履歴閲覧

### 担任
- クラス生徒の連絡帳確認・既読管理
- 早期警告アラート
  - P0: メンタル★1（即時対応）
  - P1: 3日連続メンタル低下（早期警告）
- Inbox Pattern（6カテゴリ分類）
- 担任メモ（個人メモ・学年共有メモ）
- 出席記録管理（出席、欠席、遅刻、早退）
- 反応・対応記録の編集

### 学年主任
- 学年統計、クラス比較
- 学年共有メモの閲覧・既読管理

### 校長/教頭
- 学校全体統計
- クラス健康ダッシュボード（学級閉鎖判断支援）

### システム管理者
- ユーザー・クラス管理
- 役割ベースアクセス制御
- データ管理（Django admin）

---

## 技術スタック

- Python 3.12 / Django 5.1
- PostgreSQL 16
- Bootstrap 5 / django-crispy-forms
- Docker / Docker Compose

---

## 制約・前提条件

- 本番環境: AWS（EC2, RDS, S3, CloudFront, ALB）にデプロイ済み
- メール送信: Mailpit（開発）、本番ではメール認証無効化（PoC環境のため）
- PostgreSQL必須

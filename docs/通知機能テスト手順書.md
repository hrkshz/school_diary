### テストケース2: 通知テンプレート作成

通知テンプレートは、繰り返し使う通知のひな形です。

#### 手順

1. 管理画面で「NOTIFICATIONS」→「Notification templates」をクリック

2. 右上の「ADD NOTIFICATION TEMPLATE」ボタンをクリック

3. 以下の内容を入力:

   **基本情報:**
   - **Code:** `welcome_email`
   - **Name:** `ウェルカムメール`
   - **Description:** `新規ユーザー登録時のウェルカムメール`
   - **Is active:** ✅ チェックを入れる

   **テンプレート内容:**
   - **Subject template:**
     ```
     {{ site_name }}へようこそ、{{ user.name }}様！
     ```

   - **Body template:**
     ```
     # ようこそ！

     {{ user.name }}様、{{ site_name }}へのご登録ありがとうございます。

     ## 次のステップ

     1. プロフィールを設定する
     2. 最初のプロジェクトを作成する
     3. チームメンバーを招待する

     ご不明な点がございましたら、お気軽にお問い合わせください。
     ```

   **通知設定:**
   - **Notification types:** `email` を選択（Ctrlキーを押しながらクリックで複数選択可能）
   - **Default priority:** `normal` を選択

4. 画面下部の「SAVE」ボタンをクリック

#### 期待される結果

- 「The notification template "welcome_email - ウェルカムメール" was added successfully.」というメッセージが表示される
- テンプレート一覧に戻り、作成したテンプレートが表示される

#### 確認ポイント

- テンプレート一覧で、`welcome_email - ウェルカムメール` が表示されていることを確認
- 「IS ACTIVE」列に緑のチェックマークが表示されていることを確認

### テストケース3: 通知の手動作成

実際に通知を作成してみます。

#### 手順

1. 管理画面で「NOTIFICATIONS」→「Notifications」をクリック

2. 右上の「ADD NOTIFICATION」ボタンをクリック

3. 以下の内容を入力:

   **受信者情報:**
   - **Recipient:** 先ほど作成したスーパーユーザーを選択（admin@example.com）
   - **Recipient email:** `admin@example.com` （自動入力される場合もあります）

   **通知内容:**
   - **Notification type:** `email` を選択
   - **Subject:** `テスト通知`
   - **Body:** `これはテスト通知です。`

   **優先度と状態:**
   - **Priority:** `normal` を選択
   - **Status:** `pending` （送信待ち）を選択

4. 画面下部の「SAVE」ボタンをクリック

#### 期待される結果

- 「The notification "Notification object (1)" was added successfully.」というメッセージが表示される
- 通知一覧に戻り、作成した通知が表示される

#### 確認ポイント

- **Status:** `pending` （送信待ち）になっていることを確認
- **Created at:** 現在の日時が表示されていることを確認

## Djangoシェルでの動作確認

より実践的なテストとして、Djangoシェルでプログラムから通知を作成します。

### テストケース4: Djangoシェルでの通知作成

#### 手順

1. Djangoシェルを起動

```bash
docker-compose run --rm django python manage.py shell
```

2. 以下のコードを1行ずつ入力していきます（`>>>` はプロンプトなので入力不要）

```python
# 必要なモジュールをインポート
from django.contrib.auth import get_user_model
from kits.notifications.services import NotificationService
from kits.notifications.models import NotificationTemplate, NotificationType

# Userモデルを取得
User = get_user_model()

# ユーザーを取得（先ほど作成したスーパーユーザー）
user = User.objects.get(email='admin@example.com')
print(f"ユーザー: {user.name} ({user.email})")

# テンプレートを取得
template = NotificationTemplate.objects.get(code='welcome_email')
print(f"テンプレート: {template.name}")

# 通知サービスを初期化
service = NotificationService()

# 通知を作成
notification = service.create_from_template(
    template_code='welcome_email',
    recipient=user,
    context={
        'site_name': 'LabApp',
    }
)

print(f"通知作成完了: ID={notification.pk}")
print(f"件名: {notification.subject}")
print(f"本文（最初の100文字）: {notification.body[:100]}")
```

3. シェルを終了

```python
exit()
```

#### 期待される出力例

```
ユーザー: 管理者 (admin@example.com)
テンプレート: ウェルカムメール
通知作成完了: ID=2
件名: LabAppへようこそ、管理者様！
本文（最初の100文字）: <h1>ようこそ！</h1>
<p>管理者様、LabAppへのご登録ありがとうございます。</p>
<h2>次のステップ</h2>
```

#### 確認ポイント

- エラーが発生せずに通知が作成される
- 件名にユーザー名（管理者）とサイト名（LabApp）が埋め込まれている
- 本文がMarkdownからHTMLに変換されている（`<h1>`, `<p>` タグが含まれる）

### テストケース5: メール送信テスト

作成した通知を実際に送信してみます。

#### 手順

1. Djangoシェルを起動

```bash
docker-compose run --rm django python manage.py shell
```

2. 以下のコードを入力

```python
from kits.notifications.models import Notification
from kits.notifications.services import NotificationService

# 送信待ちの通知を取得
notification = Notification.objects.filter(status='pending').first()

if notification:
    print(f"通知ID: {notification.pk}")
    print(f"件名: {notification.subject}")
    print(f"宛先: {notification.recipient_email}")

    # 通知を送信
    service = NotificationService()
    success = service.send_notification(notification)

    if success:
        print("✅ 送信成功！")
        notification.refresh_from_db()
        print(f"ステータス: {notification.status}")
        print(f"送信日時: {notification.sent_at}")
    else:
        print("❌ 送信失敗")
else:
    print("送信待ちの通知がありません")

exit()
```

#### 期待される出力例

```
通知ID: 2
件名: LabAppへようこそ、管理者様！
宛先: admin@example.com
✅ 送信成功！
ステータス: sent
送信日時: 2025-10-04 15:30:45.123456+00:00
```

#### メール確認方法

**ローカル環境（開発モード）の場合:**

メールは実際には送信されず、Djangoのコンソールに出力されます。

```bash
# Djangoコンテナのログを確認
docker-compose logs django --tail=50
```

ログに以下のようなメール内容が表示されます:

```
Content-Type: text/plain; charset="utf-8"
MIME-Version: 1.0
Content-Transfer-Encoding: 7bit
Subject: LabAppへようこそ、管理者様！
From: noreply@school_diary.local
To: admin@example.com
Date: Fri, 04 Oct 2025 15:30:45 -0000
Message-ID: <...>

# ようこそ！

管理者様、LabAppへのご登録ありがとうございます。
...
```

#### 確認ポイント

- ステータスが `pending` → `sent` に変更されている
- `sent_at` に送信日時が記録されている
- Djangoログにメール内容が出力されている

### テストケース6: 一括送信テスト

複数の通知を一度に送信します。

#### 手順

1. まず、複数の通知を作成します

```bash
docker-compose run --rm django python manage.py shell
```

```python
from django.contrib.auth import get_user_model
from kits.notifications.services import NotificationService

User = get_user_model()
service = NotificationService()
user = User.objects.get(email='admin@example.com')

# 3件の通知を作成
notifications = []
for i in range(3):
    notification = service.create_from_template(
        template_code='welcome_email',
        recipient=user,
        context={
            'site_name': f'LabApp-{i+1}',
        }
    )
    notifications.append(notification)
    print(f"通知{i+1}を作成: ID={notification.pk}")

# 一括送信
result = service.send_batch(notifications)
print(f"\n送信結果: 成功={result['success']}件, 失敗={result['failed']}件")

exit()
```

#### 期待される出力例

```
通知1を作成: ID=3
通知2を作成: ID=4
通知3を作成: ID=5

送信結果: 成功=3件, 失敗=0件
```

#### 確認ポイント

- 3件すべての通知が成功している
- 各通知のステータスが `sent` になっている

## トラブルシューティング

### Q1: 管理画面に「NOTIFICATIONS」セクションが表示されない

**原因:** アプリが登録されていない、またはマイグレーションが適用されていない

**解決方法:**

```bash
# 設定ファイルを確認
grep -A 5 "kits.notifications" config/settings/base.py

# マイグレーションを再実行
docker-compose run --rm django python manage.py migrate notifications

# Djangoサーバーを再起動
docker-compose restart django
```

---

### Q2: スーパーユーザーのパスワードを忘れた

**解決方法:**

```bash
# パスワードをリセット
docker-compose run --rm django python manage.py changepassword admin@example.com
```

新しいパスワードを2回入力すると変更されます。

---

### Q3: 通知が送信されない（ステータスが `pending` のまま）

**原因:** サービスの `send_notification()` メソッドが呼ばれていない

**解決方法:**

```bash
docker-compose run --rm django python manage.py shell
```

```python
from kits.notifications.models import Notification
from kits.notifications.services import NotificationService

notification = Notification.objects.get(id=1)  # IDを指定
service = NotificationService()
service.send_notification(notification)
```

---

### Q4: メールがコンソールに表示されない

**原因:** EMAIL_BACKENDの設定を確認

**解決方法:**

```bash
# 設定を確認
docker-compose run --rm django python manage.py shell
```

```python
from django.conf import settings
print(settings.EMAIL_BACKEND)
# 出力: django.core.mail.backends.console.EmailBackend
```

`console.EmailBackend` であれば正常です。

## はじめに

このマニュアルは、連絡帳管理システムの本番環境を第三者がテストするためのガイドです。

### テストの目的

- 本番環境が正常に動作しているか確認
- 主要機能が期待通りに動作するか検証
- セキュリティ設定が適切か確認

### 必要なもの

- PC（Windows, Mac, Linux いずれでも可）
- ブラウザ（**Chrome推奨**、Firefox・Edgeでも可）
- インターネット接続
- 30-48分の時間

### テストの進め方

1. **スモークテスト（必須）**: 全テスター必須（約9分）
2. **機能テスト**: Priority 1（高）まで実施推奨（約22分）
3. **セキュリティ・その他**: 時間があればPriority 2以降も実施（約17分）

#### SMOKE-001: システムアクセス確認

**所要時間**: 1分
**重要度**: 必須

**【テスト手順】**

1. ブラウザを開く
2. アドレスバーに以下を入力してEnterキーを押す
   ```
   https://d2wk3j2pacp33b.cloudfront.net
   ```

**【期待結果】**

✅ ページが表示される（白い画面でない）
✅ ログインページ、またはホーム画面が表示される

#### FUNC-STU-001: 連絡帳新規作成（正常系）

**所要時間**: 3分
**重要度**: 高
**Priority**: 1（高）

**【事前準備】**

- 使用アカウント: `student_1_a_01@example.com` / `password123`

**【期待結果】**

✅ 「連絡帳を提出しました」という緑色の成功メッセージが表示される
✅ ダッシュボードに戻る
✅ 今日の日付の連絡帳が一覧に表示される
✅ 入力した内容（体調・メンタル・振り返り）が正しく表示される

**【失敗時】**

- エラーメッセージの内容を記録
- スクリーンショットを撮影

✅ 「連絡帳を更新しました」というメッセージが表示される

## よくある質問（FAQ）

### Q1: ログインできない

**A**: 以下を確認してください

- メールアドレスとパスワードをコピー&ペーストで正確に入力
- パスワードは全て「password123」（小文字）
- ブラウザのシークレットモードを試す
- キャッシュをクリア（Ctrl+Shift+Delete）

### Q2: 「連絡帳を書く」ボタンが見つからない

**A**: 以下を確認してください

- 生徒アカウントでログインしているか確認（画面右上にメールアドレス表示）
- 既に今日の連絡帳を提出済みの可能性（一日一件まで）
- ページを再読み込み（F5キー）

### Q3: テストで作成したデータが見つからない

**A**: 以下を確認してください

- 正しいアカウントでログインしているか
- 日付フィルタが正しいか
- ページを再読み込み（F5キー）

### Q4: 「403 Forbidden」エラーが出た

**A**: 正常な動作です（セキュリティテストの場合）

- SEC-001, SEC-002では403エラーが期待結果です
- それ以外で出た場合はバグの可能性、スクリーンショット撮影

### Q5: テストケースの順序を変えても良いか？

**A**: 一部のテストは順序依存です

- 推奨実行順序を守ってください
- スモークテスト（SMOKE-001〜005）は必ず最初に実行
- 依存関係のあるテストは記載の順序で実施

### Q6: スクリーンショットの撮り方がわからない

**A**: OS別の方法

- **Windows**: Snipping Tool（Win+Shift+S）
- **Mac**: Command+Shift+4
- **ブラウザ拡張**: Awesome Screenshot（推奨）

### Q7: テスト中にエラーが出たらどうする？

**A**: 以下の手順で対応

1. エラーメッセージを正確にメモ
2. スクリーンショットを撮影
3. テスト結果シートに「NG」と記入
4. 次のテストに進む（一つのエラーで全体を止めない）

### Q8: テストにどれくらい時間がかかる？

**A**: 優先度により調整可能

- **最小構成**（Priority 0のみ）: 約9分
- **推奨構成**（Priority 0-1）: 約30分
- **完全版**（全テスト）: 約48分

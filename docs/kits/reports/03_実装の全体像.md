## ファイル一覧と役割

### 実装済みファイル

#### kits/reports/__init__.py
**役割:** パッケージ初期化
**内容:** 空ファイル（Pythonパッケージとして認識させるため）

#### kits/reports/apps.py
**役割:** Djangoアプリ設定
**主要クラス:**
- `ReportsConfig`: アプリ設定クラス
- `default_auto_field`: BigAutoField指定
- `name`: "kits.reports"
- `verbose_name`: "レポート"（管理画面表示名）

**なぜ必要？**
- Django がアプリとして認識するため
- INSTALLED_APPS に "kits.reports" を追加するため

#### kits/reports/models.py
**役割:** データモデル定義
**主要クラス:**

1. **ReportTemplate**
   - 役割: 再利用可能なレポートテンプレート
   - フィールド数: 15
   - 主要フィールド: code, name, chart_config, supported_formats

2. **Report**
   - 役割: 生成されたレポート
   - フィールド数: 16
   - 主要フィールド: id(UUID), template, file, status, expires_at

3. **ReportSchedule**
   - 役割: スケジュール実行設定
   - フィールド数: 13
   - 主要フィールド: template, cron_expression, send_to_users

**Choicesクラス:**
- `ReportFormat`: PDF, CSV, XLSX, HTML
- `ReportStatus`: PENDING, GENERATING, COMPLETED, FAILED
- `ChartType`: LINE, BAR, PIE, DOUGHNUT, RADAR, SCATTER, AREA

**ヘルパーメソッド:**
- `Report.mark_as_completed()`: 生成完了としてマーク
- `Report.mark_as_failed()`: 生成失敗としてマーク
- `Report.increment_download_count()`: ダウンロード回数増加

**プロパティ:**
- `Report.is_expired`: 有効期限切れか
- `Report.is_downloadable`: ダウンロード可能か

#### kits/reports/migrations/0001_initial.py
**役割:** 初期マイグレーション
**作成されたテーブル:**
- `kits_report_templates`
- `kits_reports`
- `kits_report_schedules`

**インデックス:**
- `idx_generated_by_status`: レポート一覧高速化
- `idx_created_at`: 最新レポート取得高速化
- `idx_expires_at`: クリーンアップ高速化

#### kits/reports/charts.py（未実装）
**役割:** Chart.jsラッパー

**実装予定クラス:**
- `ChartBuilder`: ビルダーパターンでグラフ設定生成
  - `line()`: 折れ線グラフ
  - `bar()`: 棒グラフ
  - `pie()`: 円グラフ
  - `doughnut()`: ドーナツグラフ
  - `to_json()`: JSON出力

- `ChartDataset`: データセット定義
- `ChartOptions`: オプション定義

**使用例（予定）:**
```python
chart = ChartBuilder.line(
    labels=['1月', '2月', '3月'],
    datasets=[{'label': '売上', 'data': [100, 200, 150]}],
    title='月次売上',
)
chart_json = chart.to_json()
```

## データフロー

### フロー1: レポート生成

```
1. ユーザーリクエスト
   ↓
2. View: レポート生成要求受付
   ↓
3. ReportService.generate_report()
   ├→ Reportオブジェクト作成（status=PENDING）
   ├→ status=GENERATINGに変更
   ├→ _execute_query(): データ取得
   ├→ DataFrame変換（pandas）
   ├→ 形式に応じて分岐:
   │   ├→ PDFExporter.export()
   │   ├→ CSVExporter.export()
   │   └→ ExcelExporter.export()
   ├→ ファイル保存
   └→ mark_as_completed()（status=COMPLETED）
   ↓
4. レポートURL返却
   ↓
5. ユーザーダウンロード
```

### フロー2: グラフ生成（テンプレート表示）

```
1. View: データ準備
   ↓
2. ChartBuilder.line()
   ├→ labels設定
   ├→ datasets設定
   ├→ options設定
   └→ to_json(): Chart.js JSON生成
   ↓
3. Template: {% render_chart chart_json %}
   ├→ <canvas>要素生成
   └→ <script>でChart.js初期化
   ↓
4. ブラウザ: Chart.jsレンダリング
```

### フロー3: スケジュール実行

```
1. Celery Beat: cron_expressionに従って起動
   ↓
2. Celeryタスク: generate_scheduled_report.delay()
   ↓
3. ReportSchedule取得
   ↓
4. ReportService.generate_report()
   ├→ レポート生成（フロー1と同じ）
   └→ ファイル保存
   ↓
5. kits.notifications連携
   ├→ send_to_usersに通知
   └→ send_to_emailsにメール送信
   ↓
6. ReportSchedule.last_run_at更新
```

## 実装の目的

業務アプリケーションで必須となるレポート機能（グラフ生成、PDF出力、CSV/Excelエクスポート）を提供する再利用可能なDjangoパッケージを実装します。

**解決する課題:**
- 残業時間の集計レポート・部門別グラフ（課題1: 残業管理）
- 成長曲線グラフ・予防接種記録の可視化（課題2: 母子手帳）
- 打撃成績・防御率・チームスタッツのグラフ化（課題3: 野球部記録）
- すべての課題で必要な印刷・エクスポート機能

**推定所要時間:** 3-4日（実働20-30時間）

| ALLOWED_FORMATS | ["pdf", "csv", "xlsx"] | 許可する出力形式 |
| CHART_LIBRARY | "chartjs" | 使用するグラフライブラリ |

### Step 3: データモデル実装

**実装モデル:**

#### 1. ReportTemplate（レポートテンプレート）
再利用可能なレポートテンプレートを管理

**主要フィールド:**
- `code`: 一意の識別子(例: monthly_overtime_report)
- `model_name`: データソースのDjangoモデル
- `chart_config`: Chart.js設定(JSON)
- `supported_formats`: 対応する出力形式(PDF, CSV, Excel)

**設計判断:**
- `code`フィールドをunique=Trueに → システム内で一意に識別
- `chart_config`をJSONField → Chart.js設定を柔軟に保存
- `supported_formats`をArrayField → 複数形式に対応

#### 2. Report（生成されたレポート）
個々のレポート生成結果を管理

**主要フィールド:**
- `id`: UUIDField(primary_key=True) ← **重要な設計判断**
- `template`: ReportTemplateへの外部キー
- `file`: レポートファイル(upload_to="reports/%Y/%m/%d/")
- `status`: 生成状態(pending/generating/completed/failed)
- `expires_at`: 有効期限

**設計判断1: なぜUUIDをPKに？**
- ❌ AutoField(連番): `/reports/1/`, `/reports/2/`でURL推測可能
- ✅ UUIDField: `/reports/a1b2c3.../`で推測不可能
- **理由**: セキュリティ向上（他人のレポートURLを推測できない）

**設計判断2: ステータス管理**
```
pending → generating → completed
          ↓ (失敗時)
        failed
```

**設計判断3: 有効期限(expires_at)**
- `RETENTION_DAYS=30`で自動削除
- ストレージ圧迫を防ぐ
- レポートファイルは容量が大きいため

#### 3. ReportSchedule（レポートスケジュール）
定期的なレポート生成を管理

**主要フィールド:**
- `template`: ReportTemplateへの外部キー
- `cron_expression`: Cron形式のスケジュール
- `send_to_users`: 送信先ユーザー(ManyToMany)
- `send_to_emails`: 送信先メールアドレス(ArrayField)

**設計判断: Cron形式の採用**
- シンプルで理解しやすい
- 既存のCelery Beatと統合可能
- 例: `0 9 1 * *` = 毎月1日の9:00AM

**実装コード:**

```python
# kits/reports/models.py（抜粋）

class Report(models.Model):
    id = models.UUIDField(
        primary_key=True,
        default=uuid.uuid4,  # UUID自動生成
        editable=False,
    )

    # ヘルパーメソッド
    def mark_as_completed(self, file_path, file_size, row_count=0):
        """生成完了としてマーク"""
        self.status = ReportStatus.COMPLETED
        self.file = file_path
        self.file_size = file_size
        self.row_count = row_count
        self.generated_at = timezone.now()
        self.save(update_fields=[...])

    @property
    def is_downloadable(self) -> bool:
        """ダウンロード可能かどうか"""
        return (
            self.status == ReportStatus.COMPLETED
            and bool(self.file)
            and not self.is_expired
        )
```

**設計パターン:**
- ヘルパーメソッド: `mark_as_completed()`, `mark_as_failed()`
- プロパティ: `is_expired`, `is_downloadable`
- → ビジネスロジックをモデル層に集約

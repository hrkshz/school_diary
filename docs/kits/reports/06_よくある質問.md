## 🔴 エラー対応

### エラー1: ReportTemplate.DoesNotExist

**エラーメッセージ**:
```
ReportTemplate matching query does not exist.
```

**原因**:
指定したテンプレートコードが存在しない

**具体例**:
```python
service.generate_report(
    template=ReportTemplate.objects.get(code='monthly_sales'),  # ← このコードのテンプレートがない
    user=user,
)
```

**解決方法**:

**方法1: テンプレートを作成**
```python
from kits.reports.models import ReportTemplate, ReportFormat

ReportTemplate.objects.create(
    code='monthly_sales',  # ← このコードで作成
    name='月次売上レポート',
    model_name='demos.SalesRecord',
    supported_formats=[ReportFormat.PDF, ReportFormat.CSV],
)
```

**方法2: コードのtypoを確認**
```python
# ❌ typo
code='montly_sales'  # ← "h" が抜けている

# ✅ 正しい
code='monthly_sales'
```

**方法3: 管理画面で確認**
```
http://localhost:8000/admin/reports/reporttemplate/
→ 登録されているテンプレートコードを確認
```

### エラー2: レポート生成が失敗する

**症状**:
- `generate_report()`を実行すると例外が発生
- レポートステータスが'failed'になる

**原因と解決方法**:

#### 原因1: model_nameが間違っている

```python
# ❌ 間違ったモデルパス
model_name='SalesRecord'  # アプリ名が抜けている

# ✅ 正しいモデルパス
model_name='demos.SalesRecord'  # アプリ名.モデル名
model_name='school_diary.users.User'
```

#### 原因2: データが存在しない

```python
# エラー詳細を確認
report = Report.objects.get(id=report_id)
print(f"エラーメッセージ: {report.error_message}")

# データ存在確認
from django.apps import apps
Model = apps.get_model('demos', 'SalesRecord')
count = Model.objects.count()
print(f"データ件数: {count}")  # 0件ならデータがない
```

#### 原因3: queryset_filterが間違っている

```python
# ❌ 間違ったフィルター
queryset_filter='year=2025'  # Q()オブジェクト必要

# ✅ 正しいフィルター
from django.db.models import Q
template.queryset_filter = Q(year=2025, status='active')
template.save()
```

### エラー3: Chart.jsグラフが表示されない

**症状**:
- テンプレートタグ `{% render_chart %}` を使っても何も表示されない
- ブラウザのコンソールにエラーが出る

**原因と解決方法**:

#### 原因1: Chart.js CDNが読み込まれていない

**確認方法**:
```html
<!-- テンプレートの<head>に以下があるか確認 -->
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0"></script>
```

**解決方法**:
```django
<!DOCTYPE html>
<html>
<head>
    <title>ダッシュボード</title>
    <!-- Chart.js CDN を追加 -->
    <script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0"></script>
</head>
<body>
    {% load report_tags %}
    {% render_chart chart_json "myChart" 400 %}
</body>
</html>
```

#### 原因2: JSONが不正

**確認方法**:
```python
# ビューで確認
chart = ChartBuilder.line(...)
chart_json = chart.to_json()
print(chart_json)  # 正しいJSON形式か確認
```

**ブラウザのコンソールで確認**:
```javascript
// F12キー → コンソールタブ
// "Uncaught SyntaxError" などのエラーがないか確認
```

#### 原因3: canvas IDの重複

```django
<!-- ❌ 同じIDを複数使用（エラー） -->
{% render_chart chart1 "myChart" 400 %}
{% render_chart chart2 "myChart" 400 %}  <!-- IDが重複！ -->

<!-- ✅ 一意なIDを使用 -->
{% render_chart chart1 "salesChart" 400 %}
{% render_chart chart2 "costChart" 400 %}
```

### エラー4: PDFが真っ白

**症状**:
- PDF生成は成功するがファイルを開くと真っ白
- またはエラー「PDFが破損しています」

**原因と解決方法**:

#### 原因1: HTMLテンプレートが見つからない

```python
# エラーログ確認
report = Report.objects.get(id=report_id)
print(report.error_message)
# → "TemplateDoesNotExist: reports/base_report.html"

# 解決方法: テンプレートを作成
# school_diary/templates/reports/base_report.html
```

#### 原因2: WEASYPRINT_BASEURLが設定されていない

```python
# config/settings/base.py に追加
WEASYPRINT_BASEURL = "file://" + str(BASE_DIR)
```

#### 原因3: WeasyPrintの依存ライブラリがない

```bash
# EC2などLinux環境で必要なパッケージ
sudo apt-get install -y \
    libpango-1.0-0 \
    libpangoft2-1.0-0 \
    libharfbuzz-subset0

# インストール確認
python -c "import weasyprint; print('OK')"
```

**代替案: ReportLabを使う**
```python
# config/settings/base.py
REPORTS_CONFIG = {
    'PDF_BACKEND': 'reportlab',  # WeasyPrintの代わり
}
```

### エラー5: Excelファイルが開けない

**症状**:
- Excelファイルをダウンロードしても開けない
- 「ファイルが破損しています」エラー

**原因と解決方法**:

#### 原因1: openpyxlがインストールされていない

```bash
# インストール確認
python -c "import openpyxl; print(openpyxl.__version__)"

# インストール
pip install openpyxl==3.1.5
```

#### 原因2: ファイル拡張子が間違っている

```python
# ❌ 拡張子が.csv（内容はExcel）
output_format=ReportFormat.XLSX
# → ファイル名は .xlsx になるべき

# ファイル名確認
print(report.file.name)
# → 'reports/2025/10/05/report_xxx.xlsx' ✅
```

#### 原因3: DataFrameが空

```python
# データ確認
import pandas as pd
df = service._execute_query(template, parameters)
print(f"行数: {len(df)}")  # 0なら空

# 解決方法: データを作成してから生成
```

## 💡 よくある質問

### Q1: グラフの色やスタイルをカスタマイズできる？

**A:** はい、Chart.jsの全機能が使えます。

```python
from kits.reports.charts import ChartBuilder

# カスタムカラー
chart = ChartBuilder.line(
    labels=['1月', '2月', '3月'],
    datasets=[
        {
            'label': '売上',
            'data': [100, 200, 150],
            'borderColor': '#4F46E5',       # 線の色
            'backgroundColor': 'rgba(79, 70, 229, 0.2)',  # 塗りつぶし
            'borderWidth': 3,                # 線の太さ
            'pointRadius': 5,                # 点のサイズ
            'tension': 0.4,                  # 曲線の滑らかさ
        }
    ],
    title='売上推移',
)

# さらに詳細なカスタマイズ
config = chart.config
config['options']['plugins']['legend']['position'] = 'bottom'
config['options']['scales']['y']['beginAtZero'] = True
```

**参考**: [Chart.js公式ドキュメント](https://www.chartjs.org/docs/latest/)

### Q2: 大量データ（10万行以上）を扱える？

**A:** はい、ただしチャンク処理の実装を推奨します。

**問題**: 全件読み込みでメモリ不足
```python
# ❌ 10万行を一度に読み込み（メモリ不足リスク）
queryset = SalesRecord.objects.all()  # 10万件
report = service.generate_report(template, user)
```

**解決方法1: iterator()を使う**
```python
# ✅ チャンク処理（1000件ずつ）
queryset = SalesRecord.objects.all().iterator(chunk_size=1000)
# → メモリ使用量を抑える
```

**解決方法2: フィルターで絞り込む**
```python
# ✅ 必要なデータだけ取得
queryset = SalesRecord.objects.filter(
    year=2025,
    department='営業部'
)  # 1万件に絞り込み
```

**解決方法3: 非同期生成**
```python
# Celeryタスクで非同期生成（推奨）
from kits.reports.tasks import generate_report_task

task = generate_report_task.delay(
    template_id=template.id,
    user_id=user.id,
)
# → バックグラウンドで生成、ユーザーを待たせない
```

### Q3: 複数ユーザーに一斉配信できる？

**A:** はい、ReportScheduleを使います。

```python
from kits.reports.models import ReportSchedule
from django.contrib.auth import get_user_model

User = get_user_model()

# スケジュール作成
schedule = ReportSchedule.objects.create(
    template=template,
    name='週次レポート配信',
    cron_expression='0 9 * * 1',  # 毎週月曜9時
    output_format='pdf',
    is_active=True,
)

# ユーザーを追加
managers = User.objects.filter(is_staff=True)
schedule.send_to_users.set(managers)

# メールアドレスを追加
schedule.send_to_emails = [
    'ceo@example.com',
    'manager@example.com',
]
schedule.save()
```

**手動で一斉送信**:
```python
# 全ユーザーに個別レポート生成
for user in User.objects.filter(is_active=True):
    report = service.generate_report(
        template=template,
        user=user,
    )
    # kits.notificationsと連携してメール送信
```

### Q4: PDFからExcelに変換できる？

**A:** 直接変換はできませんが、同じテンプレートから別形式で再生成できます。

```python
# 既存のPDFレポート
pdf_report = Report.objects.get(id=123)
template = pdf_report.template

# 同じデータでExcel版を生成
excel_report = service.generate_report(
    template=template,
    user=pdf_report.generated_by,
    output_format=ReportFormat.XLSX,
    parameters=pdf_report.parameters,  # 同じパラメータ
)

print(f"PDF: {pdf_report.file.url}")
print(f"Excel: {excel_report.file.url}")
```

### Q5: Cron式の書き方がわからない

**A:** 基本パターンを覚えれば簡単です。

**Cron式の形式**:
```
分 時 日 月 曜日
0  9  *  *  1    = 毎週月曜9時
```

**よく使うパターン**:
```python
# 毎日9時
'0 9 * * *'

# 毎週月曜9時
'0 9 * * 1'

# 毎月1日9時
'0 9 1 * *'

# 平日（月〜金）9時
'0 9 * * 1-5'

# 毎時0分（1時間ごと）
'0 * * * *'

# 15分ごと
'*/15 * * * *'
```

**ツール**: [Crontab Guru](https://crontab.guru/) で確認できます。

### Q6: レポートの有効期限を変更したい

**A:** 生成後にexpires_atを更新できます。

```python
from datetime import timedelta
from django.utils import timezone

# レポート取得
report = Report.objects.get(id=123)

# 有効期限を延長（30日→90日）
report.expires_at = timezone.now() + timedelta(days=90)
report.save()

# 無期限にする
report.expires_at = None
report.save()

# テンプレートのデフォルト有効期限を変更
template.retention_days = 90  # 90日間
template.save()
```

### Q7: ユーザーごとにレポートの権限を管理したい

**A:** generated_byフィールドで制御できます。

```python
# ビューで権限チェック
def download_report(request, report_id):
    report = get_object_or_404(Report, id=report_id)

    # 権限チェック1: 自分が生成したレポートのみ
    if report.generated_by != request.user:
        raise PermissionDenied("このレポートにアクセスできません")

    # 権限チェック2: 管理者は全て見れる
    if not (report.generated_by == request.user or request.user.is_staff):
        raise PermissionDenied("このレポートにアクセスできません")

    # ダウンロード処理
    return FileResponse(report.file.open('rb'))
```

**Django管理画面での権限**:
```python
# kits/reports/admin.py
class ReportAdmin(admin.ModelAdmin):
    def get_queryset(self, request):
        qs = super().get_queryset(request)
        # 管理者以外は自分のレポートのみ
        if not request.user.is_superuser:
            qs = qs.filter(generated_by=request.user)
        return qs
```

### Q8: グラフをPDFに埋め込める？

**A:** はい、HTMLテンプレートにChart.jsを埋め込めます。

**注意**: Chart.jsはJavaScriptなので、PDF化時にはサーバーサイドでレンダリングが必要です。

**方法1: 静的画像に変換してから埋め込む**
```python
# Chart.jsではなく、matplotlibを使う
import matplotlib.pyplot as plt

# グラフ生成
plt.plot([1, 2, 3], [100, 200, 150])
plt.savefig('chart.png')

# HTMLテンプレートで画像を埋め込む
# <img src="chart.png" />
```

**方法2: WeasyPrintでChart.jsをレンダリング**
```html
<!-- テンプレート -->
<canvas id="chart"></canvas>
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0"></script>
<script>
new Chart(document.getElementById('chart'), {{ chart_json|safe }});
</script>

<!-- WeasyPrintはJavaScript未対応のため、画像に変換が必要 -->
```

**推奨**: データテーブルとして埋め込む
```django
{% load report_tags %}
<h2>売上データ</h2>
{% render_data_table data "売上実績" %}
<!-- テーブルならPDFに正しく埋め込まれる -->
```

### Q9: テンプレート変更時に既存レポートはどうなる？

**A:** 既存レポートには影響しません（設計判断）。

```python
# テンプレート更新
template = ReportTemplate.objects.get(code='monthly_sales')
template.chart_config = {...}  # グラフ設定を変更
template.save()

# 過去のレポートは変更されない
old_report = Report.objects.filter(template=template).first()
print(old_report.file.url)  # 古い設定のまま

# 新しいレポートは新設定で生成
new_report = service.generate_report(template, user)
print(new_report.file.url)  # 新しい設定
```

**理由**: レポートは「履歴」として価値がある（[02_設計思想.md](./02_設計思想.md#重要な設計判断2-なぜtemplateon_deleteset_null) 参照）

## ⚡ パフォーマンス改善

### 大量データの最適化

**問題**: 10万件のデータでレポート生成が遅い

**解決方法1: select_relatedで最適化**
```python
# ❌ N+1問題
queryset = SalesRecord.objects.all()
for record in queryset:
    print(record.user.name)  # 毎回クエリ発行😓

# ✅ select_relatedで最適化
queryset = SalesRecord.objects.select_related('user', 'department')
for record in queryset:
    print(record.user.name)  # クエリ1回😊
```

**解決方法2: only()で必要なフィールドのみ取得**
```python
# ❌ 全フィールド取得
queryset = SalesRecord.objects.all()

# ✅ 必要なフィールドのみ
queryset = SalesRecord.objects.only('month', 'amount', 'department')
# → クエリサイズが小さくなる
```

**解決方法3: Celeryで非同期処理**
```python
# ✅ バックグラウンドで生成
from kits.reports.tasks import generate_report_task

task = generate_report_task.delay(template.id, user.id)
# → ユーザーを待たせない
```

### ファイルストレージの最適化

**問題**: レポートファイルでディスク容量が圧迫される

**解決方法1: 自動クリーンアップ**
```python
# Celeryタスクで定期削除（毎日実行）
from kits.reports.tasks import cleanup_expired_reports

cleanup_expired_reports.delay()
```

**解決方法2: 有効期限を短く設定**
```python
# テンプレートごとに設定
template.retention_days = 7  # 7日間で自動削除
template.save()
```

**解決方法3: S3などクラウドストレージを使う**
```python
# config/settings/production.py
DEFAULT_FILE_STORAGE = 'storages.backends.s3boto3.S3Boto3Storage'
AWS_STORAGE_BUCKET_NAME = 'my-reports-bucket'
```

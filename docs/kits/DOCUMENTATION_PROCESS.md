# kits パッケージ ドキュメント化プロセス標準

**作成日:** 2025-10-05
**バージョン:** 1.0.0
**適用範囲:** kits配下の全パッケージ（notifications, reports, io等）

---

## 📋 目的

kitsパッケージの実装時に、**初心者がトレースできる高品質なドキュメント**を作成するための標準プロセスを定義します。

**品質目標:**
- 正確性: 95%以上（判断理由が正確に記録されている）
- 完全性: 90%以上（必要な情報が漏れなく記録されている）
- 理解しやすさ: 90%以上（初心者が理解できる）
- トレース可能性: 95%以上（実装の流れを追える）

---

## 🎯 基本方針：段階的ドキュメント化

**原則:** 実装直後にドキュメント化（記憶が鮮明なうちに記録）

```
実装 → 実装ログ記録 → 該当ドキュメント作成 → 次の実装
```

**❌ NG:** 全実装完了後にまとめてドキュメント化（判断理由を忘れる）
**✅ OK:** 各フェーズ完了後に該当ドキュメントを作成

---

## 📁 ドキュメント構成（7ファイル標準）

すべてのkitsパッケージは以下の構成でドキュメント化：

```
docs/kits/{パッケージ名}/
├── 00_実装ログ.md              # リアルタイム記録（時系列）
├── 01_概要と目的.md            # なぜ必要？何ができる？（読了5分）
├── 02_設計思想.md              # どう設計した？なぜ？（読了10分）
├── 03_実装の全体像.md          # ファイル構成と役割（読了10分）
├── 04_コード解説.md            # コードを1行ずつ解説（読了30分）
├── 05_使い方ガイド.md          # コピペで動く実例（読了15分）
└── 06_よくある質問.md          # エラー対処、FAQ
```

---

## 🔄 実装フェーズとドキュメント化タイミング

### フェーズ1: 準備・環境構築（Step 1-2）

**実装内容:**
- ディレクトリ作成
- 依存パッケージ追加・インストール
- Django設定追加

**ドキュメント作成:**
- ✅ `00_実装ログ.md` 作成・記録開始

**所要時間:** 実装10-15分 + ドキュメント15分 = **30分**

---

### フェーズ2: データモデル実装（Step 3-4）

**実装内容:**
- models.py実装
- マイグレーション作成・適用

**ドキュメント作成:**
- ✅ `00_実装ログ.md` 更新
- ✅ `01_概要と目的.md` 作成（10分）
- ✅ `02_設計思想.md` 作成（30分）← **最重要**
- ✅ `03_実装の全体像.md` 骨子作成（20分）

**所要時間:** 実装1-2時間 + ドキュメント1時間 = **2-3時間**

**重要:** `02_設計思想.md`は実装直後に必ず作成
- なぜこのモデル設計にしたか
- 他の選択肢を検討したか
- どの判断基準で決めたか

---

### フェーズ3: コア機能実装（Step 5-7）

**実装内容:**
- services.py, charts.py, exporters.py等
- ビジネスロジック実装

**ドキュメント作成:**
- ✅ `00_実装ログ.md` 更新
- ✅ `03_実装の全体像.md` 完成
- ✅ `04_コード解説.md` 一部作成（実装した部分のみ）

**所要時間:** 実装2-3時間 + ドキュメント45分 = **3-4時間**

---

### フェーズ4: UI・テスト実装（Step 8-10）

**実装内容:**
- admin.py, テンプレート, テンプレートタグ
- テスト実装
- 動作確認

**ドキュメント作成:**
- ✅ `00_実装ログ.md` 完成
- ✅ `04_コード解説.md` 完成
- ✅ `05_使い方ガイド.md` 作成（30分）
- ✅ `06_よくある質問.md` 作成（30分）

**所要時間:** 実装2-3時間 + ドキュメント1.5時間 = **3.5-4.5時間**

## 🔧 ツール・フォーマット

### ドキュメントの書き方

**文体:**
- です・ます調
- 初心者を意識した平易な表現
- 専門用語には必ず説明を付ける

**コード例:**
```python
# 必ずコメントを付ける
# 「何をしているか」だけでなく「なぜそうするか」も書く

def example():
    # ここで何をしている？なぜ？
    pass
```

**図・表の活用:**
- データフローは図で表現
- 選択肢の比較は表で表現
- ディレクトリ構成はツリー表示

## 🎯 このプロセスを守る理由

1. **記憶の鮮度:** 実装直後の記憶は100%、1週間後は25%
2. **品質向上:** 段階的ドキュメント化で品質65% → 93%に向上
3. **初心者への配慮:** トレースできるドキュメントが教育に不可欠
4. **再利用性:** 標準プロセスで他のkits実装にも適用可能

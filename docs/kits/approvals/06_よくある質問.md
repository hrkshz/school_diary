## ❌ よくあるエラーと対処法

### エラー1: `ValueError: Only draft requests can be submitted`

**エラーメッセージ**:
```python
ValueError: Only draft requests can be submitted
```

**原因**:
既に提出済み（pending, approved, rejected, cancelled）の承認依頼を再度提出しようとしている

**対処法**:
```python
# 提出前にステータスを確認
if approval.status == "draft":
    approval = service.submit_request(approval)
else:
    print(f"すでに{approval.get_status_display()}状態です")
```

---

### エラー2: `ValueError: User XXX is not authorized to approve step YYY`

**エラーメッセージ**:
```python
ValueError: User manager@example.com is not authorized to approve step Step 2
```

**原因**:
ユーザーが承認ステップの承認者ロール（Group）に属していない

**対処法**:
```python
# ユーザーを適切なグループに追加
director_group = Group.objects.get(name="Director")
user.groups.add(director_group)

# または、ユーザーが承認可能かをチェック
if step.can_approve(user):
    service.approve_step(request, step, user)
else:
    print("承認権限がありません")
```

---

### エラー3: `ValueError: Current step is Step 1, not Step 2`

**エラーメッセージ**:
```python
ValueError: Current step is Step 1: Manager Approval, not Step 2: Director Approval
```

**原因**:
現在のステップと異なるステップを承認しようとしている

**対処法**:
```python
# 常に current_step を使う
approval = service.approve_step(
    request=approval,
    step=approval.current_step,  # 必ず current_step を使う
    approver=user,
)
```

---

### エラー4: `ApprovalRequest matching query does not exist`

**エラーメッセージ**:
```python
ApprovalRequest.DoesNotExist: ApprovalRequest matching query does not exist.
```

**原因**:
存在しない承認依頼IDを参照している

**対処法**:
```python
# get()の代わりにfilter()を使う
approval = ApprovalRequest.objects.filter(id=request_id).first()

if approval:
    # 処理
else:
    print("承認依頼が見つかりません")
```

---

### エラー5: `TypeError: User() got unexpected keyword arguments: 'username'`

**エラーメッセージ**:
```python
TypeError: User() got unexpected keyword arguments: 'username'
```

**原因**:
school_diaryのUserモデルは`email`をUSERNAME_FIELDとしており、`username`フィールドがない

**対処法**:
```python
# username ではなく email を使う
user = User.objects.create_user(
    email="user@example.com",  # username ではなく email
    password="password123",
)
```

## 🤔 FAQ

### Q1: 承認フローの途中でステップを追加できますか？

**A**: はい、可能です。ただし、**既存の承認依頼には影響しません**。

```python
# 新しいステップを追加
workflow.steps.create(
    order=3,  # 新しいステップ
    name="役員承認",
    approver_role=executive_group,
)

# 既存の承認依頼は元のフロー（Step 1→2）のまま
# 新しい承認依頼は新しいフロー（Step 1→2→3）になる
```

**既存の承認依頼にも反映させたい場合**:
管理画面で個別に`current_step`を変更するか、カスタムスクリプトで一括更新します。

---

### Q2: 承認フローをスキップできますか？

**A**: 管理画面または直接DBを操作することで可能です。

```python
# 管理者権限で強制的に承認完了にする
approval.status = "approved"
approval.current_step = None
approval.completed_at = timezone.now()
approval._history_user = admin_user
approval._change_reason = "管理者により強制承認"
approval.save()
```

---

### Q3: 承認者を動的に変更できますか？

**A**: ステップの`approver_role`を変更することで可能ですが、既存の承認依頼には影響しません。

**個別の承認依頼で承認者を変更したい場合**:
カスタムフィールドを追加する必要があります。

```python
# カスタマイズ例
class CustomApprovalRequest(ApprovalRequest):
    custom_approver = models.ForeignKey(User, null=True, ...)

    def get_pending_approvers(self):
        if self.custom_approver:
            return User.objects.filter(pk=self.custom_approver.pk)
        return super().get_pending_approvers()
```

---

### Q4: 承認期限を個別に設定できますか？

**A**: はい、承認依頼作成後に`deadline`フィールドを変更できます。

```python
# 承認依頼作成
approval = service.create_request(...)

# 期限を個別に設定
approval.deadline = timezone.now() + timezone.timedelta(hours=72)
approval.save()
```

---

### Q5: 複数の承認ワークフローを同じモデルで使えますか？

**A**: はい、可能です。

```python
# 通常の承認フロー
normal_approval = service.create_request(
    workflow=normal_workflow,
    content_object=document,
    requester=user,
)

# 緊急の承認フロー（1段階のみ）
urgent_approval = service.create_request(
    workflow=urgent_workflow,
    content_object=another_document,
    requester=user,
)
```

---

### Q6: 並列承認で「全員の承認が必要」にできますか？

**A**: はい、`required_approvals`を承認者の人数と同じにします。

```python
# レビュアーが3人いる場合
reviewer_group = Group.objects.get(name="Reviewer")
reviewers_count = reviewer_group.user_set.count()  # → 3

workflow.steps.create(
    order=1,
    name="レビュアー承認",
    approver_role=reviewer_group,
    is_parallel=True,
    required_approvals=reviewers_count,  # 全員の承認が必要
)
```

---

### Q7: 承認アクションを取り消すことはできますか？

**A**: 承認アクションは変更不可ですが、管理者が`ApprovalAction`レコードを削除し、`current_step`を前のステップに戻すことは可能です。

**推奨**:
承認を取り消す代わりに、「差戻し」機能を使って申請者に戻すことをお勧めします。

```python
# 差戻しを使う
service.return_to_requester(
    request=approval,
    step=approval.current_step,
    approver=manager,
    comment="承認を取り消します。修正して再提出してください。",
)
```

### Q8: 通知が送信されません

**A**: kits.notificationsが正しく設定されているか確認してください。

```python
# kits.notificationsが利用可能かチェック
try:
    from kits.notifications.services import NotificationService
    print("kits.notifications は利用可能です")
except ImportError:
    print("kits.notifications がインストールされていません")

# Django設定を確認
# config/settings/base.py
INSTALLED_APPS = [
    ...
    "kits.notifications.apps.NotificationsConfig",
    ...
]

NOTIFICATIONS_CONFIG = {
    "ENABLED": True,
    ...
}
```

**通知テンプレートを作成**:

```python
from kits.notifications.models import NotificationTemplate

NotificationTemplate.objects.create(
    code="approval_request_submitted",
    subject="承認依頼: {{workflow_name}}",
    body="承認依頼が提出されました。",
)

NotificationTemplate.objects.create(
    code="approval_request_pending",
    subject="承認依頼: {{workflow_name}}",
    body="承認をお願いいたします。",
)
```

### Q9: 承認フローの進行状況を表示したい

**A**: `ApprovalAction`から承認履歴を取得して表示できます。

```python
from kits.approvals.models import ApprovalAction

# 承認アクション履歴を取得
actions = ApprovalAction.objects.filter(request=approval).order_by("created_at")

# 進行状況を表示
for action in actions:
    print(f"✓ {action.step.name}: {action.get_action_display()}")
    print(f"  承認者: {action.approver.email}")
    print(f"  日時: {action.created_at}")
    print(f"  コメント: {action.comment}")
```

---

### Q10: Celeryが設定されていない場合はどうなりますか？

**A**: Celeryタスクは使えませんが、kits.approvals自体は正常に動作します。

期限チェック、リマインダー、エスカレーションが必要な場合は、Celeryの設定が必要です。

**Celeryの最小設定**:

```python
# config/settings/base.py
CELERY_BROKER_URL = "redis://localhost:6379/0"
CELERY_RESULT_BACKEND = "redis://localhost:6379/0"

# Celery Beatスケジュール
from celery.schedules import crontab

CELERY_BEAT_SCHEDULE = {
    "check-overdue-approvals": {
        "task": "kits.approvals.check_overdue_requests",
        "schedule": crontab(minute=0),  # 毎時0分
    },
}
```

### Q11: 承認ステップの順序を変更できますか？

**A**: はい、管理画面で`order`フィールドを変更できます。

**注意**: 既存の承認依頼には影響しません。

### Q12: 承認履歴をエクスポートできますか？

**A**: はい、Django管理画面または独自のスクリプトでエクスポート可能です。

```python
import csv

# 承認履歴をCSVに出力
with open("approval_history.csv", "w") as f:
    writer = csv.writer(f)
    writer.writerow(["ID", "Workflow", "Status", "Requester", "Created", "Completed"])

    for req in ApprovalRequest.objects.all():
        writer.writerow([
            req.id,
            req.workflow.name,
            req.get_status_display(),
            req.requester.email,
            req.created_at,
            req.completed_at,
        ])
```

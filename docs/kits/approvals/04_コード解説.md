## 📚 前提知識: Pythonの基礎

### 1. クラス（class）とは？

**簡単に言うと**: データと機能をまとめた「設計図」

```python
# 例: 車のクラス
class Car:
    def __init__(self, color):  # 初期化メソッド（コンストラクタ）
        self.color = color      # 属性（データ）

    def drive(self):            # メソッド（機能）
        print("走ります")
```

- `class Car:` - "Car"という設計図を作る
- `__init__` - 車を作るときに最初に実行される
- `self.color` - この車の色（属性）
- `def drive(self):` - 車の機能（メソッド）

### 2. モデル（Model）とは？

**簡単に言うと**: データベースのテーブルを表すクラス

```python
class ApprovalRequest(models.Model):
    # ↑ models.Model を継承 = データベースのテーブルになる

    status = models.CharField(...)  # テーブルの列（カラム）
```

### 3. フィールド（Field）とは？

**簡単に言うと**: テーブルの列（カラム）

```python
status = models.CharField(max_length=20)
# ↑ status という名前の文字列型のカラム（最大20文字）

amount = models.IntegerField()
# ↑ amount という名前の整数型のカラム

created_at = models.DateTimeField(auto_now_add=True)
# ↑ created_at という名前の日時型のカラム（自動で現在時刻が入る）
```

### 4. ForeignKey（外部キー）とは？

**簡単に言うと**: 他のテーブルとの関連付け

```python
workflow = models.ForeignKey(ApprovalWorkflow, ...)
# ↑ ApprovalWorkflow テーブルの id と紐付く
```

**例**:
```
ApprovalRequest テーブル:
| id | workflow_id | status  |
|----|-------------|---------|
| 1  | 5           | pending |
       ↓ (この5が ForeignKey)
ApprovalWorkflow テーブル:
| id | name      |
|----|-----------|
| 5  | 2段階承認 |
```

```python
# 通常の書き方
actions = []
for req in requests:
    actions.append(req.actions.all())
```

### パターン2: 辞書の get() メソッド

```python
# エラーになる可能性がある

# 安全
color = colors.get(obj.status, "gray")  # キーがなければ "gray"
```

### パターン3: f-string（フォーマット文字列）

```python
# 古い書き方
msg = "User %s is not authorized" % approver

msg = f"User {approver} is not authorized"
```

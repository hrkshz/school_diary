## 🤔 承認フローシステムとは？

### 簡単に言うと

**「申請→承認→完了」という流れを、どんなデータにも追加できる仕組み**です。

### 具体例

#### 例1: 残業申請
```
社員が残業を申請
  ↓
課長が承認
  ↓
部長が承認
  ↓
承認完了（残業が認められる）
```

#### 例2: 休暇申請
```
社員が休暇を申請
  ↓
マネージャーが承認
  ↓
承認完了（休暇が認められる）
```

#### 例3: 購買申請
```
担当者が備品購入を申請
  ↓
上長が承認
  ↓
経理が承認
  ↓
承認完了（購入が認められる）
```

---

## 🎯 なぜ承認フローシステムが必要なのか？

### 問題1: 同じようなコードを何度も書いている

従来の実装では、残業申請、休暇申請、購買申請など、**それぞれに承認機能を作る必要がありました**。

```python
# 残業申請モデル
class OvertimeRequest(models.Model):
    status = models.CharField(...)  # draft, pending, approved
    manager_approved = models.BooleanField(...)
    director_approved = models.BooleanField(...)
    # ... 承認ロジックを実装

# 休暇申請モデル（同じようなコードを再度書く！）
class LeaveRequest(models.Model):
    status = models.CharField(...)  # draft, pending, approved
    manager_approved = models.BooleanField(...)
    # ... 承認ロジックを実装
```

**これは非効率です。**

### 問題2: 承認フローの変更が大変

「課長承認 → 部長承認」を「課長承認のみ」に変更したい場合、**全てのモデルを修正する必要がありました**。

### 問題3: 承認の履歴が残らない

誰が、いつ、どのステップで承認したのか、**詳細な履歴を記録する仕組みがありませんでした**。

### 問題4: 期限管理ができない

承認が放置されていても気づけない、**リマインダーを送る仕組みがありませんでした**。

---

## ✨ kits.approvalsで解決できること

### 解決1: 1つの承認システムを全てのモデルで再利用

```python
# どのモデルにも承認フローを追加できる
approval = ApprovalService().create_request(
    workflow=my_workflow,
    content_object=overtime_request,  # 残業申請
    requester=user,
)

approval = ApprovalService().create_request(
    workflow=my_workflow,
    content_object=leave_request,  # 休暇申請
    requester=user,
)

approval = ApprovalService().create_request(
    workflow=my_workflow,
    content_object=purchase_request,  # 購買申請
    requester=user,
)
```

**1つの承認システムを作れば、どのデータにも使い回せます。**

### 解決2: 承認フローを管理画面で変更可能

```python
# 管理画面で承認フローを定義
workflow = ApprovalWorkflow.objects.create(name="2段階承認")
workflow.steps.create(order=1, name="課長承認", approver_role=manager_group)
workflow.steps.create(order=2, name="部長承認", approver_role=director_group)
```

**コードを変更せずに、管理画面で承認フローを追加・変更できます。**

### 解決3: 完全な承認履歴を自動記録

```python
# 承認アクションが自動で記録される
service.approve_step(request, step, approver, comment="承認します")
# → ApprovalActionに記録（誰が、いつ、どのステップで、何をしたか）
```

**django-simple-historyと連携して、完全な変更履歴を記録します。**

### 解決4: 期限管理とリマインダー

```python
# デフォルト期限を設定
workflow.default_deadline_hours = 48  # 48時間以内に承認

# Celeryタスクで自動チェック
@shared_task
def check_overdue_requests():
    # 期限切れの承認依頼をチェックして通知
```

**期限切れの自動チェック、リマインダー送信、エスカレーションができます。**

## 🚀 kits.approvalsでできること

### 1. 基本的な承認フロー

```python
# 直列承認: Step1 → Step2 → Step3
workflow = ApprovalWorkflow.objects.create(name="3段階承認")
workflow.steps.create(order=1, name="課長承認", approver_role=manager_group)
workflow.steps.create(order=2, name="部長承認", approver_role=director_group)
workflow.steps.create(order=3, name="役員承認", approver_role=executive_group)
```

### 2. 並列承認

```python
# 3人の承認者のうち2人の承認が必要
workflow.steps.create(
    order=1,
    name="レビュアー承認",
    approver_role=reviewer_group,
    is_parallel=True,
    required_approvals=2,  # 3人中2人
)
```

### 3. 自動承認

```python
# 申請者が承認者ロールに属している場合は自動承認
workflow.steps.create(
    order=1,
    name="課長承認",
    approver_role=manager_group,
    auto_approve_if_requester_in_role=True,
)
```

### 4. 承認・否認・差戻し

```python
# 承認
service.approve_step(request, step, approver, comment="承認します")

# 否認
service.reject_step(request, step, approver, comment="却下します")

# 差戻し
service.return_to_requester(request, step, approver, comment="修正してください")
```

### 5. 通知との連携（kits.notifications）

```python
# 承認依頼を提出すると、承認者に自動で通知
service.submit_request(request)
# → 承認者にメール通知

# 承認が完了すると、申請者に自動で通知
service.approve_step(request, step, approver)
# → 申請者にメール通知
```

### 6. 期限管理

```python
# 期限切れの承認依頼を取得
overdue_requests = service.get_overdue_requests()

# Celeryタスクで定期チェック
# 1時間ごとに期限切れをチェックして通知
```

### 7. ユーザーごとの承認待ちリスト

```python
# ユーザーが承認可能な保留中の承認依頼を取得
pending = service.get_pending_requests_for_user(user)
```

### 8. Django管理画面での操作

```
/admin/approvals/approvalworkflow/  # ワークフロー管理
/admin/approvals/approvalrequest/   # 承認依頼一覧
/admin/approvals/approvalaction/    # 承認アクション履歴
```

## 📊 kits.approvalsの特徴

### ✅ 汎用性
- **GenericForeignKey**: どんなモデルにも承認フローを追加可能
- **再利用可能**: 1つのワークフローを複数のモデルで共有

### ✅ 柔軟性
- **直列承認**: Step1 → Step2 → ...
- **並列承認**: N人中M人の承認が必要
- **自動承認**: 条件を満たせば自動で次のステップへ

### ✅ 完全な履歴管理
- **django-simple-history**: 全ての変更を記録
- **承認アクション**: 誰が、いつ、何をしたか

### ✅ 期限管理
- **デフォルト期限**: ワークフローごとに設定
- **期限切れチェック**: Celeryタスクで定期実行
- **リマインダー**: 期限前に自動通知

### ✅ 他kitsとの統合
- **kits.notifications**: 通知機能との連携
- **kits.audit**: 監査ログとの連携

## 🔍 他の実装との違い

### django-fsm単体

```python
# django-fsmだけでは...
class Request(models.Model):
    status = FSMField(default='draft')

    @transition(field=status, source='draft', target='pending')
    def submit(self):
        pass
```

- ❌ 承認ステップを動的に変更できない
- ❌ 複数の承認者を管理できない
- ❌ 期限管理ができない

### kits.approvals

```python
# kits.approvalsなら...
workflow = ApprovalWorkflow.objects.create(name="承認フロー")
workflow.steps.create(order=1, name="ステップ1", approver_role=group1)
workflow.steps.create(order=2, name="ステップ2", approver_role=group2)
```

- ✅ 管理画面で承認ステップを追加・変更可能
- ✅ 複数の承認者を柔軟に管理
- ✅ 期限管理、リマインダー、履歴管理が標準装備

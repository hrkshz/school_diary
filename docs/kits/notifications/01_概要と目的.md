## 🎯 なぜ通知機能が必要なのか？

### 身近な例で考える

あなたが**残業申請システム**を作っているとします。

**通知機能がない場合** 😓
```
社員: 残業申請を提出した
上司: （気づかない...）
社員: 承認されたかな？（ログインして確認）
上司: （3日後にやっと気づく）「遅くなってごめん！」
社員: 😞
```

**通知機能がある場合** 😊
```
社員: 残業申請を提出した
システム: 上司にメール送信「新しい残業申請があります」
上司: すぐに承認！
システム: 社員にメール送信「承認されました」
社員: 😊 助かった！
```

### 通知機能の重要性

業務アプリケーションでは、**ユーザーに適切なタイミングで情報を届ける**ことが非常に重要です。

**具体的な場面:**
- ✅ 申請が承認されたとき → 申請者にメール通知
- ✅ 健診の予約日が近づいたとき → リマインダー通知
- ✅ 新しいタスクが割り当てられたとき → 担当者に通知
- ✅ システムエラーが発生したとき → 管理者に緊急通知

## 📦 kits.notificationsとは？

### 簡単に言うと

業務アプリケーションで必要な**通知機能をすぐに使える形で提供する**Djangoパッケージです。

### 何が嬉しいのか？

**従来の方法（通知機能をゼロから実装）** 😓
```python
# メール送信のコードを書く（1-2時間）
def send_email(to, subject, body):
    # SMTPサーバー設定
    # メール作成
    # エラーハンドリング
    # ...100行以上のコード

# テンプレート管理のコードを書く（1-2時間）
# リトライ処理のコードを書く（1時間）
# 履歴記録のコードを書く（30分）

# 合計: 4-5時間 + デバッグ時間
```

**kits.notificationsを使う場合** 😊
```python
# たった3行！

service = NotificationService()
service.create_from_template("approval_request", user, context)

# 所要時間: 30分（設定 + 動作確認）
```

**時短効果: 3.5-4.5時間！** 🚀

## ✨ 主な機能

### 1. メール通知

**できること:**
- ユーザーにメールを送信
- HTMLメールとテキストメールの両方に対応
- テンプレートを使ったパーソナライズ（「○○さん」など）

**使用例:**
```python
# 承認申請の通知
notification = service.create_from_template(
    template_code="approval_request",
    recipient=manager,  # 上司
    context={
        "申請者": employee.name,
        "申請日": request.date,
        "理由": request.reason,
    }
)
```

### 2. システム内通知

**できること:**
- アプリ内で通知を表示（ベルアイコンなど）
- 未読・既読の管理
- 通知の履歴保存

**イメージ:**
```
🔔 3件の未読通知
  ├─ 残業申請が承認されました (2時間前)
  ├─ 新しいタスクが割り当てられました (1日前)
  └─ 健診の予約日が近づいています (3日前)
```

### 3. テンプレート管理

**できること:**
- 通知の「型」を事前に作成
- 管理画面から簡単に編集
- 同じ内容の通知を何度も使い回せる

**具体例:**

**テンプレート作成（1回だけ）:**
```
コード: "welcome_email"
件名: "ようこそ、{{user.name}}さん！"
本文:
  こんにちは、{{user.name}}さん。
  アカウント登録ありがとうございます。
  ログインURLは以下です：
  {{login_url}}
```

**使用（何度でも）:**
```python
# 太郎さんに送信
service.create_from_template("welcome_email", taro, context)

# 花子さんに送信
service.create_from_template("welcome_email", hanako, context)

# 1000人に送信
for user in users:
    service.create_from_template("welcome_email", user, context)
```

### 4. 非同期送信（Celery連携）

**できること:**
- メール送信を裏で実行（ユーザーを待たせない）
- 大量送信でもシステムが重くならない
- 送信失敗時の自動リトライ

**イメージ:**
```
ユーザー: 申請ボタンをクリック
システム: 「申請を受け付けました」（即座に表示）
        ↓
      （裏でメール送信処理が動く）
        ↓
      メール送信完了！
```

### 5. リマインダー（スケジュール送信）

**できること:**
- 指定した日時に通知を送信
- 定期的な通知（毎週月曜、毎月1日など）
- 期限前の自動リマインダー

**使用例:**
```python
# 3日後に健診リマインダーを送信
notification = service.create_from_template(
    template_code="health_checkup_reminder",
    recipient=user,
    context={"健診日": checkup_date},
)
notification.scheduled_at = checkup_date - timedelta(days=3)
notification.save()
```

## 📊 技術的な特徴（簡単に）

### 初心者でも安心

- 特殊な設定不要
- Djangoを知っていれば使える

✅ **テストが充実**
- 全11テストが成功（2025-10-05時点）
- バグが少ない

✅ **ドキュメント完備**
- このドキュメントシリーズで全て説明

### 本格的な機能も搭載

✅ **Markdown対応**
- テンプレートでMarkdown記法が使える
- 自動的にHTMLメールに変換

✅ **XSS対策済み**
- 悪意のあるHTMLを自動除去
- セキュリティ面も安心

✅ **リトライ機能**
- 送信失敗時に自動リトライ（最大3回）
- 一時的なエラーにも対応

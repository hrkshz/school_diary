### 2.1 NotificationTemplateRenderer

#### render_subject（件名レンダリング）

```python
@classmethod
def render_subject(cls, template_str: str, context: Dict[str, Any]) -> str:
    """
    件名をレンダリング

    Args:
        template_str: テンプレート文字列
        context: コンテキストデータ

    Returns:
        レンダリング済みの件名
    """
    template = Template(template_str)
    rendered = template.render(Context(context))
    # 件名は1行のみ、余分な空白を削除
    return rendered.strip().replace('\n', ' ')
```

**初心者向け解説**:
```python
@classmethod
# ↑ クラスメソッド（インスタンス不要で呼び出せる）
# NotificationTemplateRenderer.render_subject(...) ← 直接呼べる

def render_subject(cls, template_str, context):
# cls = クラス自身（NotificationTemplateRenderer）
# template_str = "{{user.name}}さん、承認されました"
# context = {"user": user_object}

    template = Template(template_str)
    # ↑ Django Templateオブジェクトを作成

    rendered = template.render(Context(context))
    # ↑ テンプレートをレンダリング
    # Context(context) = 辞書をContextオブジェクトに変換

    return rendered.strip().replace('\n', ' ')
    # strip() = 前後の空白・改行を削除
    # replace('\n', ' ') = 改行をスペースに置換
    # 理由: メール件名は1行である必要がある
```

**具体例**:
```python
# 入力

context = {"user": User(name="太郎")}

# 処理
rendered = template.render(Context(context))
# → "太郎さん、\n承認されました"

result = rendered.strip().replace('\n', ' ')
# → "太郎さん、 承認されました"

# なぜ必要？
# メールの件名に改行が入るとエラーになる場合がある
```

#### render_body（本文レンダリング）

```python
@classmethod
def render_body(cls, template_str: str, context: Dict[str, Any],
               use_markdown: bool = True) -> str:
    """本文をレンダリング"""
    template = Template(template_str)
    rendered = template.render(Context(context))

    if use_markdown:
        # MarkdownをHTMLに変換
        html = markdown.markdown(
            rendered,
            extensions=['extra', 'nl2br', 'sane_lists']
        )
        # HTMLをサニタイズ(XSS対策)
        safe_html = bleach.clean(
            html,
            tags=cls.ALLOWED_TAGS,
            attributes=cls.ALLOWED_ATTRIBUTES,
            strip=True
        )
        return safe_html

    return rendered
```

**Markdown変換の解説**:
```python
# 入力（Markdown）
rendered = """
## 承認のお知らせ

こんにちは、{{user.name}}さん。

**承認**されました。
"""

# markdown.markdown(...) で変換
html = """
<h2>承認のお知らせ</h2>
<p>こんにちは、太郎さん。</p>
<p><strong>承認</strong>されました。</p>
"""

# extensions の意味:
# 'extra'      → テーブル、脚注などの拡張構文
# 'nl2br'      → 改行を<br>に変換
# 'sane_lists' → リストの挙動を改善
```

**bleach.clean（XSS対策）の解説**:
```python
# 悪意のあるHTML
html = '<script>alert("XSS")</script><p>本文</p>'

# bleach.clean で安全化
safe_html = bleach.clean(
    html,
    tags=["p", "strong", "em"],  # 許可するタグ
    attributes={},                 # 許可する属性
    strip=True                     # 不許可タグは削除
)
# 結果: '<p>本文</p>'
# <script>タグは削除された！

# なぜ必要？
# ユーザーが悪意のあるコードを入力しても、
# それがそのままHTMLメールに入らないようにするため
```

**ALLOWED_TAGSの定義**:
```python
ALLOWED_TAGS = [
    'a', 'abbr', 'acronym', 'b', 'blockquote', 'code',
    'em', 'i', 'li', 'ol', 'p', 'strong', 'ul', 'h1',
    'h2', 'h3', 'h4', 'h5', 'h6', 'br', 'div', 'span',
]

# 許可: <p>, <strong>, <h2> など一般的なタグ
# 不許可: <script>, <iframe>, <object> など危険なタグ

# メールで使う程度のHTML装飾は許可、
# JavaScriptなど危険なものは拒否
```

### 2.2 NotificationService

#### create_from_template（テンプレートから通知作成）

```python
def create_from_template(
    self,
    template_code: str,
    recipient: 'User',
    context: Optional[Dict[str, Any]] = None,
    notification_type: str = NotificationType.EMAIL,
    scheduled_at: Optional[timezone.datetime] = None,
) -> Notification:
    """テンプレートから通知を作成"""

    # 1. テンプレート取得
    template = NotificationTemplate.objects.get(code=template_code)

    # 2. コンテキスト準備
    if context is None:
        context = {}
    context.setdefault("recipient", recipient)

    # 3. レンダリング
    subject = NotificationTemplateRenderer.render_subject(
        template.subject_template, context
    )
    body = NotificationTemplateRenderer.render_body(
        template.body_template, context
    )

    # 4. JSONシリアライズ可能なcontextを作成
    serializable_context = {
        k: v for k, v in context.items()
        if not hasattr(v, '_meta')  # Djangoモデルを除外
    }

    # 5. 通知作成
    notification = Notification.objects.create(
        recipient=recipient,
        template=template,
        notification_type=notification_type,
        subject=subject,
        body=body,
        context_data=serializable_context,
        scheduled_at=scheduled_at,
    )

    return notification
```

**ステップごとの解説**:

**ステップ1: テンプレート取得**
```python
template = NotificationTemplate.objects.get(code=template_code)

# code="welcome_email" のテンプレートを取得
# 存在しない場合: NotificationTemplate.DoesNotExist例外が発生
```

**ステップ2: コンテキスト準備**
```python
if context is None:
    context = {}
# ↑ contextが渡されない場合、空の辞書を作成

context.setdefault("recipient", recipient)
# ↑ "recipient"キーがなければ追加
# setdefault = 既に存在する場合は上書きしない

# なぜ必要？
# テンプレートで {{recipient.name}} を使いたい場合に便利
```

**ステップ4: JSONシリアライズ対策**
```python
serializable_context = {
    k: v for k, v in context.items()
    if not hasattr(v, '_meta')  # Djangoモデルを除外
}

# 問題:
context = {"user": user_object}  # Djangoモデルを含む
notification.context_data = context  # JSONFieldに保存
# ↓
# TypeError: Object of type User is not JSON serializable

# 解決:
# Djangoモデル(_meta属性を持つ)を除外
# context_data には文字列、数値、リストなどのみ保存
```

## 3️⃣ tasks.py - 非同期処理

### 3.1 send_notification_task

```python
@shared_task(bind=True, max_retries=3)
def send_notification_task(self, notification_id: int):
    """通知を送信する非同期タスク"""
    try:
        notification = Notification.objects.get(id=notification_id)

        # ステータスを送信中に
        notification.status = NotificationStatus.SENDING
        notification.save(update_fields=["status"])

        # メール送信
        send_mail(
            subject=notification.subject,
            message=notification.body,
            from_email=settings.DEFAULT_FROM_EMAIL,
            recipient_list=[notification.recipient.email],
            html_message=notification.html_body or None,
        )

        # 送信成功
        notification.mark_as_sent()

    except Exception as exc:
        # 送信失敗
        notification.mark_as_failed(str(exc))
        # リトライ
        raise self.retry(exc=exc, countdown=60)
```

**@shared_task の解説**:
```python
@shared_task(bind=True, max_retries=3)

# @shared_task = Celeryタスクとして登録
# bind=True = selfを渡す（リトライに必要）
# max_retries=3 = 最大3回までリトライ

# なぜ非同期？
# メール送信は5-10秒かかる → ユーザーを待たせたくない

# 同期処理（悪い）:
def send_notification(notification_id):
    send_mail(...)  # 5秒待つ😓
    return "完了"

# 非同期処理（良い）:
send_notification_task.delay(notification_id)  # 0.1秒で返る😊
# 裏でCeleryワーカーが実行
```

**リトライの仕組み**:
```python
except Exception as exc:
    notification.mark_as_failed(str(exc))
    raise self.retry(exc=exc, countdown=60)

# リトライフロー:
# 1回目: 送信失敗 → 60秒後にリトライ
# 2回目: 送信失敗 → 60秒後にリトライ
# 3回目: 送信失敗 → 60秒後にリトライ
# 4回目: max_retries=3 → リトライ終了

# countdown=60 = 60秒待ってからリトライ
# 理由: 一時的なネットワークエラーの回復を待つ
```

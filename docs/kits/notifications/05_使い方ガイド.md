## 🎯 基本的な使い方

### 例1: シンプルな通知送信

**ユースケース**: ユーザー登録時に歓迎メールを送る

#### ステップ1: 通知テンプレートを作成

**方法A: 管理画面で作成（初心者向け）**

```
1. http://localhost:8000/admin/ にアクセス
2. 「通知テンプレート」→「追加」
3. 以下を入力:
   - テンプレートコード: welcome_email
   - テンプレート名: ウェルカムメール
   - 件名テンプレート: ようこそ、{{user.get_full_name}}さん！
   - 本文テンプレート: （下記参照）
   - 通知タイプ: email にチェック
4. 保存
```

本文テンプレート:
```markdown
# ようこそ！

{{user.get_full_name}}さん、school_diaryへのご登録ありがとうございます。

## 次のステップ

1. プロフィールを設定する
2. 最初のプロジェクトを作成する
3. チームメンバーを招待する

ご不明な点がございましたら、お気軽にお問い合わせください。
```

**方法B: Pythonコードで作成（上級者向け）**

```python
from kits.notifications.models import NotificationTemplate, NotificationType

# テンプレート作成（既に存在する場合はスキップ）
template, created = NotificationTemplate.objects.get_or_create(
    code='welcome_email',
    defaults={
        'name': 'ウェルカムメール',
        'subject_template': 'ようこそ、{{user.get_full_name}}さん！',
        'body_template': '''
# ようこそ！

{{user.get_full_name}}さん、school_diaryへのご登録ありがとうございます。

## 次のステップ

1. プロフィールを設定する
2. 最初のプロジェクトを作成する
3. チームメンバーを招待する

ご不明な点がございましたら、お気軽にお問い合わせください。
        ''',
        'notification_types': [NotificationType.EMAIL],
    }
)

if created:
    print("テンプレート作成しました！")
else:
    print("テンプレートは既に存在します")
```

#### ステップ2: 通知を送信

```python
from django.contrib.auth import get_user_model
from kits.notifications.services import NotificationService

User = get_user_model()

# サービスインスタンス作成
service = NotificationService()

# ユーザー取得
user = User.objects.get(username="taro")

# 通知作成＆送信
notification = service.create_from_template(
    template_code='welcome_email',
    recipient=user,
    context={
        'site_name': 'school_diary',
    }
)

# 即座に送信（同期処理）
service.send_notification(notification)
print(f"通知を送信しました: {notification.subject}")
```

#### ステップ3: 非同期送信（推奨）

```python
from kits.notifications.tasks import send_notification_task

# 通知作成
notification = service.create_from_template(
    template_code='welcome_email',
    recipient=user,
    context={'site_name': 'school_diary'}
)

# Celeryタスクで非同期送信
send_notification_task.delay(notification.id)
print("通知をキューに追加しました（裏で送信されます）")
```

### 例2: 承認依頼通知

**ユースケース**: 残業申請が提出されたら、上司に承認依頼メールを送る

#### テンプレート作成

```python
from kits.notifications.models import NotificationTemplate, NotificationType

template, created = NotificationTemplate.objects.get_or_create(
    code='approval_request',
    defaults={
        'name': '承認依頼',
        'subject_template': '【要承認】{{requester.get_full_name}}さんから{{request_title}}',
        'body_template': '''
# 承認依頼

{{approver.get_full_name}}様

{{requester.get_full_name}}から承認依頼が届いています。

## 申請内容

**タイトル:** {{request_title}}

**申請日:** {{request_date}}

**内容:**
{{request_description}}

**承認期限:** {{deadline}}

[承認画面を開く]({{approval_url}})
        ''',
        'notification_types': [NotificationType.EMAIL, NotificationType.IN_APP],
    }
)
```

#### 通知送信（実際のアプリケーションでの使用例）

```python
# apps/overtime/views.py

from kits.notifications.services import NotificationService
from kits.notifications.tasks import send_notification_task

class OvertimeRequestCreateView(CreateView):
    """残業申請作成ビュー"""

    def form_valid(self, form):
        # 申請を保存
        overtime_request = form.save(commit=False)
        overtime_request.requester = self.request.user
        overtime_request.save()

        # 承認者に通知
        service = NotificationService()
        notification = service.create_from_template(
            template_code='approval_request',
            recipient=overtime_request.approver,  # 上司
            context={
                'approver': overtime_request.approver,
                'requester': self.request.user,
                'request_title': '残業申請（10月分）',
                'request_date': overtime_request.created_at.strftime('%Y-%m-%d'),
                'request_description': overtime_request.reason,
                'deadline': overtime_request.deadline.strftime('%Y-%m-%d'),
                'approval_url': self.request.build_absolute_uri(
                    reverse('overtime:approval', args=[overtime_request.id])
                ),
            }
        )

        # 非同期送信
        send_notification_task.delay(notification.id)

        messages.success(self.request, "申請を提出しました。承認者に通知を送信しました。")
        return super().form_valid(form)
```

### 例3: スケジュール送信（リマインダー）

**ユースケース**: 健診の3日前にリマインダーを送る

```python
from datetime import datetime, timedelta
from kits.notifications.services import NotificationService

# テンプレート作成（省略）
# code='health_checkup_reminder'

# 健診予約データ
checkup_date = datetime(2025, 10, 10, 9, 0)  # 2025年10月10日 9:00
reminder_date = checkup_date - timedelta(days=3)  # 3日前

# 通知作成
service = NotificationService()
notification = service.create_from_template(
    template_code='health_checkup_reminder',
    recipient=user,
    context={
        'checkup_date': checkup_date.strftime('%Y年%m月%d日 %H:%M'),
        'location': '本社ビル3F 健康管理室',
    }
)

# スケジュール設定
notification.scheduled_at = reminder_date
notification.save()

print(f"リマインダーを{reminder_date}に送信予定で登録しました")
```

#### スケジュール送信の仕組み

```python
# Celery Beatで定期的に実行されるタスク
from kits.notifications.tasks import send_scheduled_notifications

# このタスクが毎分実行され、scheduled_at が過ぎた通知を送信
# celery beat スケジュール設定は config/settings/base.py に記載
```

### 例4: 一括送信

**ユースケース**: 返却期限が近い図書館利用者全員にリマインダーを送る

```python
from datetime import datetime, timedelta
from kits.notifications.services import NotificationService

# 返却期限が3日以内のユーザーを取得（仮のデータ）
due_soon_users = [
    {
        'user': User.objects.get(username='taro'),
        'books': [
            {'title': 'Djangoの教科書', 'due_date': '2025-10-07'},
            {'title': 'Pythonクックブック', 'due_date': '2025-10-08'},
        ],
    },
    {
        'user': User.objects.get(username='hanako'),
        'books': [
            {'title': 'データベース入門', 'due_date': '2025-10-06'},
        ],
    },
    # ... 他のユーザー
]

# 一括通知作成
service = NotificationService()
notifications = []

for data in due_soon_users:
    notification = service.create_from_template(
        template_code='library_return_reminder',
        recipient=data['user'],
        context={'books': data['books']}
    )
    notifications.append(notification)

print(f"{len(notifications)}件の通知を作成しました")

# バッチ送信
result = service.send_batch(notifications)
print(f"送信成功: {result['success']}件、失敗: {result['failed']}件")
```

## 🔧 高度な使い方

### ユーザーの未読通知を表示

```python
from kits.notifications.services import NotificationService

# ビューで使用
def notification_list_view(request):
    service = NotificationService()

    # 未読数取得
    unread_count = service.get_unread_count(request.user)

    # 未読通知一覧
    unread_notifications = service.get_user_notifications(
        request.user,
        status="sent",  # 送信済み（=未読）
        limit=10
    )

    context = {
        'unread_count': unread_count,
        'notifications': unread_notifications,
    }
    return render(request, 'notifications/list.html', context)
```

### 通知を既読にする

```python
from kits.notifications.models import Notification

# 個別に既読
notification = Notification.objects.get(id=123)
notification.mark_as_read()

# 全て既読
service = NotificationService()
service.mark_all_as_read(user)
```

### 通知タイプを選択

```python
from kits.notifications.models import NotificationType

# メール通知のみ
notification = service.create_from_template(
    template_code='welcome_email',
    recipient=user,
    context={},
    notification_type=NotificationType.EMAIL  # ← 指定
)

# アプリ内通知のみ
notification = service.create_from_template(
    template_code='approval_request',
    recipient=user,
    context={},
    notification_type=NotificationType.IN_APP  # ← 指定
)
```

### 優先度を設定

```python
from kits.notifications.models import NotificationPriority

# 緊急通知
notification = service.create_from_template(
    template_code='emergency_alert',
    recipient=admin_user,
    context={},
    priority=NotificationPriority.URGENT  # ← 緊急
)

# 通常の通知
notification = service.create_from_template(
    template_code='info_update',
    recipient=user,
    context={},
    priority=NotificationPriority.NORMAL  # ← 通常（デフォルト）
)
```

## 🧪 Djangoシェルで試す

```bash
# Djangoシェル起動
docker-compose -f docker-compose.local.yml run --rm django python manage.py shell

# またはローカルの場合
python manage.py shell
```

```python
# Pythonシェル内で実行

from django.contrib.auth import get_user_model
from kits.notifications.services import NotificationService
from kits.notifications.models import NotificationTemplate, NotificationType

User = get_user_model()

# 1. テンプレート作成
template, created = NotificationTemplate.objects.get_or_create(
    code='test_notification',
    defaults={
        'name': 'テスト通知',
        'subject_template': 'こんにちは、{{user.username}}さん',
        'body_template': 'これはテスト通知です。',
        'notification_types': [NotificationType.EMAIL],
    }
)
print(f"テンプレート: {template.code}")

# 2. ユーザー取得（または作成）
user, created = User.objects.get_or_create(
    username='testuser',
    defaults={'email': 'test@example.com'}
)
print(f"ユーザー: {user.username}")

# 3. 通知作成
service = NotificationService()
notification = service.create_from_template(
    template_code='test_notification',
    recipient=user,
    context={}
)
print(f"通知作成: {notification.subject}")

# 4. 確認
print(f"ステータス: {notification.status}")
print(f"受信者: {notification.recipient.username}")
print(f"件名: {notification.subject}")
print(f"本文: {notification.body}")

# 5. 送信（開発環境ではコンソールにメールが出力される）
service.send_notification(notification)
print("送信完了！")

# 6. ステータス確認
notification.refresh_from_db()
print(f"送信後のステータス: {notification.status}")
```

## 💡 ベストプラクティス

### 1. テンプレートコードの命名規則

```python
# ✅ 良い例: 意味が分かる命名
code='approval_request'
code='welcome_email'
code='password_reset'
code='health_checkup_reminder'

# ❌ 悪い例: 意味不明な命名
code='notif1'
code='template_a'
code='test'
```

### 2. コンテキストの設計

```python
# ✅ 良い例: 明確なキー名
context={
    'approver': approver_user,
    'requester': requester_user,
    'request_title': '残業申請',
    'request_date': '2025-10-05',
    'approval_url': 'https://...',
}

# ❌ 悪い例: 不明確なキー名
context={
    'u1': user1,
    'u2': user2,
    'title': 'xxx',
    'date': 'xxx',
    'url': 'xxx',
}
```

### 3. 非同期送信を優先

```python
# ✅ 良い例: 非同期送信（ユーザーを待たせない）
notification = service.create_from_template(...)
send_notification_task.delay(notification.id)
return redirect('success')  # すぐにリダイレクト

# ❌ 悪い例: 同期送信（ユーザーが待つ）
notification = service.create_from_template(...)
service.send_notification(notification)  # 5秒待つ😓
return redirect('success')
```

### 4. エラーハンドリング

```python
# ✅ 良い例: 例外を処理
from kits.notifications.models import NotificationTemplate

try:
    notification = service.create_from_template(
        template_code='welcome_email',
        recipient=user,
        context={}
    )
    send_notification_task.delay(notification.id)
    messages.success(request, "通知を送信しました")
except NotificationTemplate.DoesNotExist:
    messages.error(request, "通知テンプレートが見つかりません")
    logger.error("Template 'welcome_email' does not exist")
except Exception as e:
    messages.error(request, "通知の送信に失敗しました")
    logger.exception("Failed to send notification", exc_info=e)
```

## 📝 まとめ

### 基本的な流れ

1. **テンプレート作成** - 管理画面またはコード
2. **通知作成** - `service.create_from_template()`
3. **送信** - `send_notification_task.delay()` （非同期推奨）

### よく使うコード

```python
# 通知送信の基本形
from kits.notifications.services import NotificationService
from kits.notifications.tasks import send_notification_task

service = NotificationService()
notification = service.create_from_template(
    template_code='テンプレートコード',
    recipient=user,
    context={'キー': '値'}
)
send_notification_task.delay(notification.id)
```

**これだけ覚えれば、ほとんどのケースに対応できます！**

## 🎯 なぜデータインポート機能が必要なのか？

### 身近な例で考える

あなたが**図書館システム**を作っているとします。

**インポート機能がない場合** 😓
```
管理者: 1000冊の本のデータをExcelで受け取った
管理者: （1冊ずつ手動で登録...）
管理者: 「疲れた...タイプミスもあるかも」
管理者: 😞 3日かかった
```

**インポート機能がある場合** 😊
```
管理者: 1000冊の本のデータをExcelで受け取った
管理者: システムにファイルをアップロード
システム: 自動で1000冊を登録、エラーは赤字で表示
管理者: 😊 3分で完了！
```

### データインポート機能の重要性

業務アプリケーションでは、**大量のデータを効率的に取り込む**ことが非常に重要です。

**具体的な場面:**
- ✅ 既存システムからのデータ移行（5校の図書館データを統合）
- ✅ 定期的なデータ更新（毎月の売上データCSVを取り込み）
- ✅ 外部システムとの連携（他社からのExcelデータ取り込み）
- ✅ 初期データの一括登録（新規開設時の商品マスタ登録）

## 📦 kits.ioとは？

### 簡単に言うと

**kits.io**は、業務アプリケーションで必要な**データインポート/エクスポート機能**を
提供する再利用可能なDjangoパッケージです。

```
CSV/TSV/Excelファイル
         ↓
   [kits.io]
    ・読み込み
    ・バリデーション
    ・重複チェック
    ・新規採番
         ↓
   Djangoデータベース
```

### 正式な定義

- **正式名称**: kits.io (Data Import/Export Kit)
- **タイプ**: 再利用可能なDjangoアプリケーション
- **目的**: CSV/TSV/Excelファイルのインポート/エクスポート機能を提供
- **対象**: 業務Webアプリケーション全般

## 💡 何ができるのか？

### 主要機能一覧

#### 1. **ファイル形式対応** 📄

```python
# CSV形式
importer = CSVImporter(model_name='library.Book')
history = importer.import_file('books.csv')

# TSV形式（タブ区切り）
importer = TSVImporter(model_name='library.Book')
history = importer.import_file('books.tsv')

# Excel形式
importer = ExcelImporter(model_name='library.Book')
history = importer.import_file('books.xlsx')
```

**対応形式:**
- ✅ CSV（カスタム区切り文字対応）
- ✅ TSV（タブ区切り）
- ✅ Excel（.xlsx、シート指定可能）

#### 2. **文字コード自動検出** 🌏

```python
# Shift-JISやEUC-JPなどを自動検出
importer = CSVImporter(
    model_name='library.Book',
    auto_detect_encoding=True  # 自動検出
)
```

**対応文字コード:**
- ✅ UTF-8
- ✅ Shift-JIS（日本語Windows）
- ✅ EUC-JP（Unix系）
- ✅ その他（chardetで検出可能な全形式）

#### 3. **重複チェック・新規採番** 🔄

**課題例**: 5つの学校図書館のデータを統合する際、バーコードIDが重複している

```python
# 重複を自動で新規採番
importer = ExcelImporter(
    model_name='library.Book',
    unique_fields=['barcode_id'],
    duplicate_strategy='renumber'  # 重複時は新規採番
)
# 元: ABC-001 → 新: ABC-001_00001
```

**4つの重複処理戦略:**
- **skip**: 重複をスキップ（デフォルト）
- **update**: 既存レコードを更新
- **renumber**: 新規採番（連番付加）
- **error**: エラーとして扱う

#### 4. **バリデーション** ✅

```python
# Djangoモデルのバリデーションを自動適用
importer = CSVImporter(
    model_name='library.Book',
    validate=True  # バリデーション有効
)
# ISBNの形式チェック、必須フィールドチェックなど
```

**バリデーション内容:**
- ✅ Djangoモデルのバリデーションルール
- ✅ 必須フィールドチェック
- ✅ 型チェック（数値、日付など）
- ✅ カスタムバリデーション追加可能

#### 5. **詳細なインポート履歴** 📊

```python
history = importer.import_file('books.csv')

print(f"総行数: {history.total_rows}")
print(f"成功: {history.success_count}")
print(f"失敗: {history.failed_count}")
print(f"スキップ: {history.skipped_count}")
print(f"更新: {history.updated_count}")
print(f"新規採番: {history.renumbered_count}")
print(f"成功率: {history.success_rate}%")

# エラー詳細も記録
for error in history.error_details:
    print(f"行{error['row']}: {error['field']} - {error['error']}")
```

**記録される情報:**
- ✅ 統計情報（成功、失敗、スキップ、更新、新規採番）
- ✅ 成功率
- ✅ 処理時間
- ✅ 行ごとのエラー詳細
- ✅ 実行者、実行日時

#### 6. **マッピング設定** 🗺️

**課題**: 毎回同じマッピングを書くのは面倒...

```python
# 1. マッピング設定を作成（1回だけ）
mapping = ImportMapping.objects.create(
    code='book_import',
    name='図書インポート',
    model_name='library.Book',
    field_mapping={
        'タイトル': 'title',
        '著者': 'author',
        'ISBN': 'isbn',
    },
    unique_fields=['isbn'],
    duplicate_strategy='skip',
)

# 2. 以降はコードを指定するだけ
importer = CSVImporter(
    model_name='library.Book',
    mapping_code='book_import'  # 簡単！
)
```

#### 7. **大量データ対応** 🚀

```python
# チャンク処理で100万行でもOK
importer = CSVImporter(
    model_name='library.Book',
    chunk_size=1000  # 1000行ずつ処理
)
```

**特徴:**
- ✅ メモリ効率的なチャンク処理
- ✅ トランザクション管理
- ✅ 途中エラーでも処理継続


### 課題4: 図書館システム（5校のTSVインポート、バーコードID重複）

**要件**:
- 5つの学校図書館データをUTF-8 TSVで受け取る
- バーコードIDが重複している場合は新規採番

```python
# A校のデータ
importer = TSVImporter(
    model_name='library.Book',
    unique_fields=['barcode_id'],
    duplicate_strategy='renumber',  # 重複時は新規採番
    encoding='utf-8',
)
importer.import_file('school_a_books.tsv')

# B校のデータ（同じ設定）
importer.import_file('school_b_books.tsv')
# ... 以下、C校、D校、E校も同様
```

**結果:**
- ✅ 5校のデータを統合
- ✅ 重複したバーコードIDは自動で新規採番
- ✅ エラー行は記録され、修正後に再インポート可能

## 🔧 他のツールとの比較

### Django Import/Export

**Django Import/Export（サードパーティ）**:
- ✅ 管理画面から直接インポート可能
- ❌ 文字コード自動検出なし
- ❌ 重複時の新規採番機能なし
- ❌ インポート履歴の詳細記録なし

**kits.io**:
- ✅ 文字コード自動検出
- ✅ 重複時の新規採番（renumber戦略）
- ✅ 詳細なインポート履歴
- ✅ チャンク処理でメモリ効率的
- ❌ 管理画面からの直接インポートは未実装（将来拡張）

### pandas直接利用

**pandas直接利用**:
- ✅ 柔軟なデータ処理
- ❌ Djangoモデルとの統合が手間
- ❌ バリデーションを自分で実装
- ❌ 履歴管理を自分で実装

**kits.io**:
- ✅ Djangoモデルとシームレス統合
- ✅ バリデーション自動
- ✅ 履歴管理自動
- ✅ 使い方が簡単

## 🙋 よくある質問

**Q1: CSV以外の形式も対応していますか？**

A: はい、CSV/TSV/Excelに対応しています。将来的にXML/JSONも追加予定です。

**Q2: 100万行のデータでも大丈夫ですか？**

A: はい、チャンク処理（1000行ずつ）でメモリ効率的に処理できます。

**Q3: Excelのシートを指定できますか？**

A: はい、`sheet_name='シート名'`または`sheet_name=0`（インデックス）で指定できます。

**Q4: 文字化けしたCSVはどうすればいいですか？**

A: `auto_detect_encoding=True`で自動検出できます。または`encoding='shift-jis'`で明示的に指定してください。

**Q5: 重複データをスキップしたいです**

A: `duplicate_strategy='skip'`（デフォルト）で重複をスキップできます。

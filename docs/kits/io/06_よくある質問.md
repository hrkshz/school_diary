## 基本的な使い方

### Q1. CSV以外の形式も対応していますか？

**A:** はい、以下の形式に対応しています：

- ✅ CSV（カスタム区切り文字対応）
- ✅ TSV（タブ区切り）
- ✅ Excel（.xlsx、シート指定可能）

将来的にXML/JSONも追加予定です。

---

### Q2. 100万行のデータでも大丈夫ですか？

**A:** はい、チャンク処理（デフォルト1000行ずつ）でメモリ効率的に処理できます。

```python
# チャンクサイズを調整可能
importer = CSVImporter(
    model_name='library.Book',
    chunk_size=500,  # 500行ずつ処理
    user=user,
)
```

**推奨設定**:
- 10万行未満: chunk_size=1000（デフォルト）
- 10万〜100万行: chunk_size=500
- 100万行以上: chunk_size=100

---

### Q3. Excelのシートを指定できますか？

**A:** はい、シート名またはインデックスで指定できます。

```python
# シート名で指定
importer = ExcelImporter(
    model_name='library.Book',
    sheet_name='2023年度',
    user=user,
)

# インデックスで指定（0が最初のシート）
importer = ExcelImporter(
    model_name='library.Book',
    sheet_name=0,
    user=user,
)
```

---

### Q4. 文字化けしたCSVはどうすればいいですか？

**A:** `auto_detect_encoding=True`で自動検出できます。

```python
# 方法1: 自動検出（推奨）
importer = CSVImporter(
    model_name='library.Book',
    auto_detect_encoding=True,  # Shift-JIS等を自動検出
    user=user,
)

# 方法2: 明示的に指定
importer = CSVImporter(
    model_name='library.Book',
    encoding='shift-jis',  # 手動指定
    auto_detect_encoding=False,
    user=user,
)
```

**対応文字コード**:
- UTF-8
- Shift-JIS（日本語Windows）
- EUC-JP（Unix系）
- その他（chardetで検出可能な全形式）

---

### Q5. 重複データをスキップしたいです

**A:** `duplicate_strategy='skip'`（デフォルト）で重複をスキップできます。

```python
importer = CSVImporter(
    model_name='library.Book',
    unique_fields=['isbn'],  # ISBNで重複チェック
    duplicate_strategy='skip',  # 重複はスキップ
    user=user,
)

history = importer.import_file('books.csv')
print(f"スキップ: {history.skipped_count}件")
```

## エラー対処

### エラー1: `ModuleNotFoundError: No module named 'kits.io'`

**原因**: Djangoの設定でINSTALLED_APPSに追加されていない

**解決方法**:
```python
# config/settings/base.py
INSTALLED_APPS = [
    # ...
    "kits.io",  # これを追加
]
```

---

### エラー2: `ValueError: モデルが見つかりません: library.Book`

**原因**: `model_name`の指定が間違っている

**解決方法**:
```python
# ❌ 間違い
model_name='Book'  # アプリ名がない

# ✅ 正しい
model_name='library.Book'  # アプリ名.モデル名
```

**確認方法**:
```python
# Django Shellで確認
from django.apps import apps
model = apps.get_model('library', 'Book')
print(model)  # <class 'library.models.Book'>
```

---

### エラー3: `UnicodeDecodeError: 'utf-8' codec can't decode`

**原因**: ファイルの文字コードがUTF-8ではない

**解決方法**:
```python
# 自動検出を有効化
importer = CSVImporter(
    model_name='library.Book',
    auto_detect_encoding=True,  # 自動検出
    user=user,
)
```

**手動で確認する方法**:
```python
# ファイルの文字コードを確認
import chardet

with open('books.csv', 'rb') as f:
    result = chardet.detect(f.read())
    print(result)
    # {'encoding': 'shift-jis', 'confidence': 0.99}
```

---

### エラー4: `FileNotFoundError: [Errno 2] No such file or directory`

**原因**: ファイルパスが間違っている

**解決方法**:
```python
# ❌ 相対パスは避ける
importer.import_file('books.csv')

# ✅ 絶対パスを使う
import os
file_path = os.path.join('/path/to', 'books.csv')
importer.import_file(file_path)

# ✅ Djangoのプロジェクトルートからのパス
from django.conf import settings
file_path = os.path.join(settings.BASE_DIR, 'data', 'books.csv')
importer.import_file(file_path)
```

---

### エラー5: `ValidationError: {'field': ['エラーメッセージ']}`

**原因**: データがDjangoモデルのバリデーションに失敗

**解決方法**:

**方法1**: バリデーションエラーを確認
```python
history = importer.import_file('books.csv')

# エラー詳細を確認
if history.error_details:
    for error in history.error_details:
        print(f"行{error['row']}: {error['field']} - {error['error']}")
```

**方法2**: バリデーションを一時的に無効化して確認
```python
importer = CSVImporter(
    model_name='library.Book',
    validate=False,  # バリデーション無効化
    user=user,
)
history = importer.import_file('books.csv')
```

**方法3**: CSVデータを修正
```csv
# ❌ 間違い: 必須フィールドが空
title,,isbn

# ✅ 正しい: 全フィールドに値
title,author,isbn
Djangoの教科書,山田太郎,978-4-1234-5678-9
```

---

### エラー6: `IntegrityError: duplicate key value violates unique constraint`

**原因**: 重複データを挿入しようとしている

**解決方法**:
```python
# 重複チェックを有効化
importer = CSVImporter(
    model_name='library.Book',
    unique_fields=['isbn'],  # 重複チェック対象
    duplicate_strategy='skip',  # スキップ
    user=user,
)
```

## パフォーマンス

### Q6. インポートが遅いです

**A:** 以下の方法で高速化できます：

**方法1**: チャンクサイズを調整
```python
# チャンクサイズを大きくする（メモリに余裕がある場合）
importer = CSVImporter(
    model_name='library.Book',
    chunk_size=5000,  # デフォルトは1000
    user=user,
)
```

**方法2**: バリデーションを無効化（推奨しない）
```python
# バリデーションをスキップ（高速化するが危険）
importer = CSVImporter(
    model_name='library.Book',
    validate=False,  # バリデーション無効
    user=user,
)
```

**方法3**: インデックスを一時的に削除
```sql
-- PostgreSQLの場合
DROP INDEX idx_book_isbn;  -- インポート前
-- インポート実行
CREATE INDEX idx_book_isbn ON books(isbn);  -- インポート後
```

---

### Q7. メモリ不足エラーが出ます

**A:** チャンクサイズを小さくしてください。

```python
importer = CSVImporter(
    model_name='library.Book',
    chunk_size=100,  # 小さくする
    user=user,
)
```

**目安**:
- メモリ 2GB: chunk_size=100
- メモリ 4GB: chunk_size=500
- メモリ 8GB以上: chunk_size=1000（デフォルト）

## 設計・実装

### Q8. 複数のCSVを一度にインポートできますか？

**A:** ループ処理で実現できます。

```python
import glob

# ディレクトリ内の全CSVファイルをインポート
csv_files = glob.glob('/path/to/*.csv')

for csv_file in csv_files:
    importer = CSVImporter(
        model_name='library.Book',
        user=user,
    )
    history = importer.import_file(csv_file)
    print(f"{csv_file}: {history.success_count}件")
```

### Q10. インポート履歴を削除したい

**A:** Django管理画面またはコードから削除できます。

**方法1**: Django管理画面
```
管理画面 → Data Import/Export → インポート履歴 → 削除
```

**方法2**: コード
```python
from kits.io.models import ImportHistory

# 古い履歴を削除（30日以前）
from datetime import datetime, timedelta

cutoff_date = datetime.now() - timedelta(days=30)
ImportHistory.objects.filter(created_at__lt=cutoff_date).delete()
```

### Q11. 非同期でインポートできますか？

**A:** Celeryと組み合わせることで可能です（将来実装予定）。

**現在の実装**:
```python
# 同期実行（処理が完了するまで待つ）
history = importer.import_file('books.csv')
```

**Celeryを使った非同期実装**:
```python
# tasks.py
from celery import shared_task
from kits.io.importers import CSVImporter

@shared_task
def async_import_file(file_path, model_name):
    importer = CSVImporter(model_name=model_name)
    history = importer.import_file(file_path)
    return str(history.id)

# views.py
from .tasks import async_import_file

def upload_view(request):
    # ファイルアップロード後
    task = async_import_file.delay('/path/to/file.csv', 'library.Book')
    return JsonResponse({'task_id': task.id})
```

### Q12. カラム名が一致しない場合は？

**A:** `mapping`パラメータで対応関係を指定してください。

```python
# CSVのカラム名: title_ja, author_ja, isbn_ja
# Djangoモデル: title, author, isbn

importer = CSVImporter(
    model_name='library.Book',
    mapping={
        'title_ja': 'title',
        'author_ja': 'author',
        'isbn_ja': 'isbn',
    },
    user=user,
)
```

### デバッグ1: インポート履歴を確認

```python
from kits.io.models import ImportHistory

# 最新のインポート履歴を取得
history = ImportHistory.objects.latest('created_at')

print(f"ID: {history.id}")
print(f"ファイル名: {history.original_filename}")
print(f"ステータス: {history.get_status_display()}")
print(f"総行数: {history.total_rows}")
print(f"成功: {history.success_count}")
print(f"失敗: {history.failed_count}")
print(f"成功率: {history.success_rate}%")

# エラー詳細
if history.error_details:

```

## 🎓 ベストプラクティス

### ベストプラクティス1: マッピング設定を再利用

---

### ベストプラクティス2: エラーハンドリング

```python
try:
    history = importer.import_file('books.csv')

    if history.status == 'completed':
        logger.info(f"インポート成功: {history.success_count}件")
    elif history.status == 'partial':
        logger.warning(f"一部失敗: 失敗{history.failed_count}件")
        # エラー詳細をログ出力
        for error in history.error_details:
            logger.error(f"行{error['row']}: {error['error']}")
    else:
        logger.error(f"インポート失敗: {history.error_message}")

except Exception as e:
    logger.exception(f"予期しないエラー: {e}")
```

### ベストプラクティス3: トランザクション管理

```python
from django.db import transaction

with transaction.atomic():
    # インポート実行
    history = importer.import_file('books.csv')

    # エラーがあればロールバック
    if history.status == 'failed':
        raise Exception("インポート失敗")
```

### 3. `validators.py` - バリデーター（143行）

**役割**: インポートデータのバリデーション

#### 3.1 ImportValidator - Django フォームベース

**何をする？**
- Djangoモデルのバリデーションルールを自動適用
- フィールドごとのエラーを収集

```python
class ImportValidator:
    def __init__(self, model):
        # Djangoモデルフォームを動的生成
        self.form_class = type(
            f'{model.__name__}ImportForm',
            (ModelForm,),
            {'Meta': type('Meta', (), {'model': model, 'fields': '__all__'})}
        )

    def validate_row(self, row_dict):
        """1行をバリデーション"""
        form = self.form_class(data=row_dict)
        if form.is_valid():
            return {}
        return form.errors  # {'field': ['error1', 'error2']}
```

**使用例**:
```python
validator = ImportValidator(Book)

# 1行をバリデーション
errors = validator.validate_row({'title': '', 'isbn': 'invalid'})
# → {'title': ['この項目は必須です'], 'isbn': ['無効な形式']}
```

## 🔄 データフロー

### インポート処理の流れ

```
1. ユーザーがファイルを指定
   ↓
2. Importer（CSVImporter等）がインスタンス化
   ↓
3. import_file() 実行
   ├─ ImportHistory作成（status=pending）
   ├─ status=processing に更新
   ├─ ファイル読み込み（pandas）
   ├─ マッピング適用（カラム名変更）
   ├─ チャンク処理開始
   │  ├─ トランザクション開始
   │  ├─ 1行ずつ処理
   │  │  ├─ バリデーション（ImportValidator）
   │  │  ├─ 重複チェック
   │  │  │  ├─ skip → スキップ
   │  │  │  ├─ update → 更新
   │  │  │  ├─ renumber → 新規採番
   │  │  │  └─ error → エラー
   │  │  └─ Djangoモデルに保存
   │  └─ トランザクションコミット
   ├─ status=completed/failed/partial に更新
   └─ ImportHistoryを返す
   ↓
4. 結果表示（管理画面 or CLI）
```

## 🗄️ データベーステーブル

### kits_import_mappings

```sql
CREATE TABLE kits_import_mappings (
    id SERIAL PRIMARY KEY,
    code VARCHAR(100) UNIQUE NOT NULL,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    model_name VARCHAR(200) NOT NULL,
    field_mapping JSONB DEFAULT '{}',
    validation_rules JSONB DEFAULT '{}',
    unique_fields JSONB DEFAULT '[]',
    duplicate_strategy VARCHAR(20) DEFAULT 'skip',
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP NOT NULL,
    updated_at TIMESTAMP NOT NULL,
    created_by_id INTEGER REFERENCES users(id)
);
```

### kits_import_histories

```sql
CREATE TABLE kits_import_histories (
    id UUID PRIMARY KEY,
    mapping_id INTEGER REFERENCES kits_import_mappings(id),
    imported_by_id INTEGER REFERENCES users(id),
    original_filename VARCHAR(255) NOT NULL,
    file VARCHAR(100),
    file_size BIGINT DEFAULT 0,
    encoding VARCHAR(50) DEFAULT 'utf-8',
    model_name VARCHAR(200) NOT NULL,
    parameters JSONB DEFAULT '{}',
    status VARCHAR(20) DEFAULT 'pending',
    total_rows INTEGER DEFAULT 0,
    success_count INTEGER DEFAULT 0,
    failed_count INTEGER DEFAULT 0,
    skipped_count INTEGER DEFAULT 0,
    updated_count INTEGER DEFAULT 0,
    renumbered_count INTEGER DEFAULT 0,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP NOT NULL,
    updated_at TIMESTAMP NOT NULL,
    error_message TEXT,
    error_details JSONB DEFAULT '[]'
);

-- インデックス
CREATE INDEX idx_imported_by_status ON kits_import_histories(imported_by_id, status);
```

## トラブルシューティング

### ドキュメントが品質チェックで不合格の場合

**設計思想のキーワード不足**:
1. `docs/kits/{パッケージ名}/02_設計思想.md` を開く
2. 各主要判断について以下を追加：
   - なぜその設計を選んだのか（理由）
   - 他にどんな選択肢を検討したのか（代替案）
   - トレードオフは何か（得たもの・失ったもの）

**初心者が理解できない**:
1. 問題箇所を特定（どこがわかりにくいか）
2. 具体例を追加
3. 専門用語に説明を追加
4. 段階的な説明に変更

**コード例が動かない**:
1. 実際に Django shell でコード例を実行
2. エラーを修正
3. 動作確認済みコードをドキュメントに反映

### テストが失敗する場合

1. エラーメッセージを確認
2. テストコードを見直し
3. モデル/サービスを修正
4. マイグレーション再実行

# Part 1: システム概要

> **所要時間**: 5 分
> **目的**: システムの全体像を理解する

---

## 1.1 システムの目的

### 解決する課題

中学校において、生徒の日々の体調やメンタル状態を把握することは重要ですが、以下の課題がありました。

**従来の課題**:

- 紙の連絡帳では情報共有が困難
- 生徒のメンタル変化を見逃しやすい
- 担任の負担が大きい（毎日 30 名以上の確認）
- 長期的な記録の保管・検索が困難

### このシステムができること

**連絡帳管理システム**は、これらの課題に対応します。

**生徒のメンタル変化を早期発見**

- 3 日連続でメンタルが低下している生徒を自動検知し、担任に通知
- メンタル ★1 の生徒を最優先で表示（Inbox Pattern: P0 重要）

**担任の負担軽減**

- 優先度別に自動分類（P0 重要 → P3 完了済み）することで、重要な生徒から確認可能
- カード形式で一目で状況把握、ワンクリックで既読処理

**情報共有の改善**

- 学年主任や校長/教頭も統計情報を閲覧でき、学校全体で生徒を支援
- 学年共有メモで担任間の情報共有を促進

**長期記録の保管**

- デジタル化により、過去の記録を簡単に検索・参照

## 1.2 5 つの役割（ロール）

このシステムは 5 つの役割に対応しています。

| 役割             | 主な業務                                             | アクセス範囲     | ログイン後の画面                    |
| ---------------- | ---------------------------------------------------- | ---------------- | ----------------------------------- |
| **🎓 生徒**      | 連絡帳の作成・提出                                   | 自分の連絡帳のみ | 生徒ダッシュボード                  |
| **👨‍🏫 担任**      | 連絡帳確認、既読処理、担任メモ作成、出席記録         | 担当クラスのみ   | 担任ダッシュボード（Inbox Pattern） |
| **👔 学年主任**  | 学年統計、クラス比較、学年共有メモ確認               | 担当学年のみ     | 学年統計ダッシュボード              |
| **🏫 校長/教頭** | 学校全体統計、クラス健康状態確認（学級閉鎖判断支援） | 全学年           | 学校統計ダッシュボード              |
| **👤 管理者**    | ユーザー管理、クラス管理、システム設定               | 全データ         | Django 管理画面                     |

### 役割の関係図

```
管理者（全データ管理）
    ↓
校長/教頭（全学年統計）
    ↓
学年主任（学年統計）
    ↓
担任（クラス管理）
    ↓
生徒（連絡帳作成）
```

### ログインリダイレクトロジック

ログイン後、役割に応じて自動的に適切な画面にリダイレクトされます。

| 優先順位 | 条件                                   | リダイレクト先                                     |
| -------- | -------------------------------------- | -------------------------------------------------- |
| 1        | `is_superuser=True`                    | `/admin/` （Django 管理画面）                      |
| 2        | `role='school_leader'`                 | `/diary/school-overview/` （学校統計）             |
| 3        | `role='grade_leader'`                  | `/diary/grade-overview/` （学年統計）              |
| 4        | `role='teacher'` または 担任として登録 | `/diary/teacher/dashboard/` （担任ダッシュボード） |
| 5        | 上記以外（生徒）                       | `/diary/student/dashboard/` （生徒ダッシュボード） |

## 2.1 生徒向け操作

### 2.1.1 連絡帳の作成・提出

#### 手順

1. **ログイン**: 生徒用メールアドレス（例: `student_1_a_01@example.com`）とパスワード（`password123`）でログイン

2. 生徒ダッシュボードが表示されます

3. 「今日の連絡帳を書く」ボタンをクリック

4. 以下の項目を入力します：

**入力項目**:

| 項目                  | 説明                                   | 選択肢/入力例                              |
| --------------------- | -------------------------------------- | ------------------------------------------ |
| **📅 記載日**         | 前登校日の日付を選択                   | 日付ピッカーで選択                         |
| **😊 体調**           | 今日の体調を ★1〜★5 で選択             | ★1: とても悪い 〜 ★5: とても良い           |
| **😊 メンタル**       | 今日の気分を ★1〜★5 で選択             | ★1: とても落ち込んでいる 〜 ★5: とても元気 |
| **✍️ 今日の振り返り** | 今日あったこと、感じたことを自由に記入 | 「数学のテストで 85 点取れました！」       |

**★ の意味**:

- ★1: とても悪い
- ★2: 悪い
- ★3: 普通
- ★4: 良い
- ★5: とても良い

5. 「📝 提出する」ボタンをクリック

6. 「連絡帳を作成しました。」と表示されれば成功です

#### 注意事項

- **1 日 1 件まで**: 1 日に作成できる連絡帳は 1 件のみです（システム制約）
- **編集可能**: 担任が既読にする前であれば編集できます
- **既読後は編集不可**: 担任が既読にした後は編集できません（過去記録として確定）

---

### 2.1.2 連絡帳の編集

#### 手順

1. 生徒ダッシュボードで「過去の記録」をクリック

2. 過去の連絡帳一覧が表示されます

3. **未既読の連絡帳**に「✏️ 編集」ボタンが表示されます

4. 編集画面で内容を修正

5. 「📝 提出する」ボタンをクリック

#### 注意事項

- **既読前のみ編集可能**: 担任が既読にした後は編集できません

---

### 2.1.3 過去の連絡帳閲覧

#### 手順

1. 生徒ダッシュボードで「過去の記録」リンクをクリック

2. 過去の連絡帳一覧が表示されます

3. 日付、体調、メンタル、担任からの反応が確認できます

#### 表示内容

| 項目               | 説明                         |
| ------------------ | ---------------------------- |
| **日付**           | 連絡帳を提出した日           |
| **体調**           | ★1〜★5                       |
| **メンタル**       | ★1〜★5                       |
| **振り返り**       | 当日記入した内容             |
| **担任からの反応** | 担任が選択した反応（絵文字） |

---

### 2.1.4 担任からの反応確認

担任が連絡帳を確認すると、以下のような反応が表示されます。

| 反応 | 意味           | 使用場面                   |
| ---- | -------------- | -------------------------- |
| 📖   | 読んだよ       | 確認のみ（デフォルト）     |
| 👍   | いいね         | ポジティブなフィードバック |
| 💯   | よくできました | 良い内容の連絡帳           |
| 💪   | がんばったね   | 頑張りを認める             |
| 🌟   | 素晴らしい     | 特に良い内容               |
| ❤️   | 応援してるよ   | 励ましが必要な場合         |

## 2.2 担任向け操作

### 2.2.1 担任ダッシュボード概要

#### Inbox Pattern（5段階+サブセクション 自動分類）

担任ダッシュボードは、担当クラスの生徒の連絡帳を**優先度順**に自動分類して表示します。

| カテゴリ               | 優先度 | 説明                                               | カード色            | 対応目安         |
| ---------------------- | ------ | -------------------------------------------------- | ------------------- | ---------------- |
| **P0: 重要**           | 最高   | メンタル ★1（即座の対応が必要）                    | 🔴 赤               | 即座             |
| **P1: 要注意**         | 高     | 3 日連続メンタル低下（注意が必要）                 | 🟡 黄               | 当日中           |
| **P1.5: 要対応タスク** | 高     | 対応記録が設定され、対応待ちの連絡帳               | 🟠 オレンジ         | 計画的に         |
| **P2: 未対応**         | 中     | 日常タスク                                         | 🔵 青               | 時間があるときに |
| └ P2-1: 未提出         | 中     | 昨日の連絡帳が未提出の生徒                         | 🔵 青（折りたたみ） | 必要時           |
| └ P2-2: 未読           | 中     | 提出済みだがまだ確認していない連絡帳               | 🔵 青               | 時間があるときに |
| **P3: 完了済み**       | 低     | 既読・反応済みの連絡帳（前登校日以降のみ表示）     | 🟢 緑（折りたたみ） | 完了             |

#### 推奨確認順序

1. **P0: 重要** → 即座に対応（声をかける、面談）
2. **P1: 要注意** → 当日中に対応（様子確認）
3. **P1.5: 要対応タスク** → 計画的に対応（保護者連絡、個別面談）
4. **P2-2: 未読** → 時間があるときに確認
5. **P2-1: 未提出** → 必要に応じて声かけ

---

### 2.2.2 既読処理の 3 つの方法

担任ダッシュボードのカードには**3 つのボタン**があります。それぞれの使い方を説明します。

#### 方法 1: ✓ 既読ボタン（デフォルト「読みました」）

**用途**: 特に対応不要な連絡帳をすぐに既読にする

**手順**:

1. 担任ダッシュボードのカードで「✓ 既読」ボタンをクリック
2. 自動的に以下が設定されます：
   - 既読フラグ: `is_read = True`
   - 既読者: 現在のログインユーザー
   - 既読日時: 現在時刻
   - 生徒への反応: 「📖 読んだよ」（デフォルト）
   - 対応状況: `action_status = NOT_REQUIRED`（対応不要）

**特徴**:

- **最速で既読処理**できます（ワンクリック）
- 生徒には「📖 読んだよ」が表示されます
- 対応不要として記録されます

---

#### 方法 2: 📋 タスクボタン（対応記録を選択）

**用途**: 連絡帳を既読にし、かつ対応が必要なタスクとして記録する

**手順**:

1. 担任ダッシュボードのカードで「📋 タスク」ボタンをクリック
2. モーダルが表示されます
3. 「対応記録」を選択します：

**対応記録の選択肢**:

| 選択肢          | 使用場面             |
| --------------- | -------------------- |
| ⚠️ 要フォロー   | 後日確認が必要       |
| 🔴 緊急対応必要 | 即座の対応が必要     |
| 📞 保護者連絡   | 保護者への連絡が必要 |
| 🗣️ 個別面談     | 生徒との面談が必要   |
| 👥 学年共有     | 学年全体で情報共有   |
| 📝 継続観察     | 継続的な観察が必要   |

4. 「タスク化」ボタンをクリック
5. 自動的に以下が設定されます：
   - 既読フラグ: `is_read = True`
   - 既読者: 現在のログインユーザー
   - 既読日時: 現在時刻
   - 生徒への反応: 「📖 読んだよ」（デフォルト）
   - 対応記録: 選択した内容（例: `internal_action = parent_contacted`）
   - 対応状況: `action_status = IN_PROGRESS`（対応中）

**特徴**:

- 既読処理と同時に**タスク管理**ができます
- タスク化された連絡帳は **P1.5: 要対応タスク** セクションに表示されます
- 生徒には対応記録は表示されません（先生のみ）

---

#### 方法 3: 📖 詳細ボタン（生徒詳細ページで反応を選択）

**用途**: 生徒の過去履歴を確認しながら、詳細に反応を選択する

**手順**:

1. 担任ダッシュボードのカードで「📖 詳細」ボタンをクリック
2. 生徒詳細ページが表示されます
3. 過去の連絡帳履歴、担任メモを確認できます
4. 「既読にする」ボタンをクリック
5. モーダルが表示されます
6. 以下を選択します：

**生徒への反応**（生徒に表示される）:

| 選択肢            | 使用場面                   |
| ----------------- | -------------------------- |
| 📖 読んだよ       | 確認のみ（デフォルト）     |
| 👍 いいね         | ポジティブなフィードバック |
| 💯 よくできました | 良い内容の連絡帳           |
| 💪 がんばったね   | 頑張りを認める             |
| 🌟 素晴らしい     | 特に良い内容               |
| ❤️ 応援してるよ   | 励ましが必要な場合         |

**対応記録**（先生のみに表示）:

| 選択肢          | 使用場面             |
| --------------- | -------------------- |
| 記録のみ        | 特に対応不要         |
| ⚠️ 要フォロー   | 後日確認が必要       |
| 🔴 緊急対応必要 | 即座の対応が必要     |
| 📞 保護者連絡   | 保護者への連絡が必要 |
| 🗣️ 個別面談     | 生徒との面談が必要   |
| 👥 学年共有     | 学年全体で情報共有   |
| 📝 継続観察     | 継続的な観察が必要   |

7. 「保存」ボタンをクリック

**特徴**:

- **過去履歴を確認**しながら反応を選択できます
- **生徒への反応を細かく選択**できます（6 種類の絵文字）
- 対応記録も同時に設定できます

### 2.2.3 担任メモの作成・共有

#### 概要

担任メモは生徒についての長期的な観察記録や引継ぎ情報を記録する機能です。

#### メモの種類

| 種類             | 説明                         | 閲覧範囲           |
| ---------------- | ---------------------------- | ------------------ |
| **個人メモ**     | 自分だけが見るメモ           | 作成した担任のみ   |
| **学年共有メモ** | 学年の担任全員で共有するメモ | 同じ学年の担任全員 |

---

#### 手順（個人メモ）

1. 担任ダッシュボードで生徒の「📖 詳細」ボタンをクリック

2. 生徒詳細ページが表示される

3. 「メモを追加」ボタンをクリック

4. メモ内容を入力

5. 「学年会議で共有」のチェックは**オフ**のまま

6. 「保存」ボタンをクリック

---

#### 手順（学年共有メモ）

1. 担任ダッシュボードで生徒の「📖 詳細」ボタンをクリック

2. 生徒詳細ページが表示される

3. 「メモを追加」ボタンをクリック

4. メモ内容を入力

5. **「学年会議で共有」のチェックをオンにする**

6. 「保存」ボタンをクリック

---

#### 共有メモの既読処理

学年共有メモは、他の担任が作成したメモを既読処理できます。

**手順**:

1. 担任ダッシュボードで「📢 学年共有アラート」セクションを確認

2. 未読の共有メモが表示される（過去 3 日以内）

3. メモ内容を確認

### 2.5.7 テストデータ作成

#### 概要

管理画面からテストデータを作成する機能です。

**生成されるデータ**（デフォルト設定）:

- 管理者: 1 名（スーパーユーザー、削除されない）
- 校長: 1 名
- 学年主任: 3 名（1-3 年）
- 担任: 9 名（9 クラス分）
- 生徒: 270 名（各クラス 30 名 × 9 クラス）
- クラス: 9 クラス（1-3 年 × A-C クラス）
- 日記: 過去 30 日分（提出率 80%、約 6,500 件）
- 特別パターン: P0/P1/P1.5/P3（オプション）

**カスタマイズ可能な項目**:

- 既存データをクリア（スーパーユーザーは保護）
- 日記作成日数（1〜30 日）
- 生徒数/クラス（1〜30 名）
- 特別パターンを含めるかどうか

**注意事項**:

- 既存のユーザー・クラスはスキップされます（差分追加モード）
- 全ユーザーのパスワード: `password123`
- 実行時間: 約 1〜2 分
- スーパーユーザーは削除されません（安全）

---

#### 手順

1. **管理者でログイン**: `admin@example.com` / `password123`

   - Django 管理画面のホームから「テストデータ作成」リンクをクリック

3. **設定画面で内容を指定**:

   - 既存データをクリア: チェックすると全データ削除（スーパーユーザーは保護）
   - 日記作成日数: 1〜30 日（デフォルト: 30 日）
   - 生徒数/クラス: 1〜30 名（デフォルト: 30 名）
   - 特別パターンを含める: チェックすると P0/P1 等を作成

   - バックグラウンドでテストデータ生成コマンドが実行されます
   - ローディング画面が表示されます

6. **実行完了を確認**:
   - 約 1〜2 分待ちます
   - 完了画面に統計情報が表示されます
   - 「ユーザー一覧を見る」ボタンで作成されたユーザーを確認できます

## 3.1 よくある質問

### Q1. 連絡帳を間違えて提出してしまいました。修正できますか？

**A1.** 担任が既読にする前であれば、生徒ダッシュボードから「過去の記録」→「✏️ 編集」ボタンをクリックして修正できます。既読後は修正できません。

---

### Q2. 1 日に 2 回連絡帳を提出できますか？

**A2.** いいえ、1 日 1 件までです。システムの制約として、同じ日付で複数の連絡帳を作成することはできません。

---

### Q3. 体調やメンタルの ★ の意味がわかりません。

**A3.** ★1〜★5 で選択します。

- ★1: とても悪い
- ★2: 悪い
- ★3: 普通
- ★4: 良い
- ★5: とても良い

---

### Q4. 担任からの反応がまだ表示されません。

**A4.** 担任が連絡帳を確認するまで反応は表示されません。通常、当日または翌日には確認されます。

---

### Q5. 未読の連絡帳が多すぎて確認しきれません。優先順位はありますか？

**A5.** はい、担任ダッシュボードは優先度順に表示されています。まず「P0: 重要」セクションから確認してください。メンタル ★1 や 3 日連続低下の生徒は即座の対応が必要です。

---

### Q6. 学年共有メモと個人メモの違いは何ですか？

**A6.**

- **個人メモ**: 自分だけが見るメモ。他の担任には表示されません。
- **学年共有メモ**: 同じ学年の担任全員が見るメモ。学年会議で情報共有する際に使用します。

---

### Q7. 対応記録を間違えて選択してしまいました。修正できますか？

**A7.** はい、生徒の連絡帳詳細ページから再度「既読にする」ボタンをクリックして、正しい対応記録を選択し直してください。

---

### Q8. 出席記録を間違えて入力しました。修正できますか？

**A8.** はい、担任ダッシュボードの「📝 出席入力」ボタンをクリックし、正しい出席状況を選択し直してください。

---

### Q9. 他のクラスの生徒の連絡帳も見れますか？

**A9.** いいえ、担任は自分が担当するクラスの生徒の連絡帳のみ確認できます。これはプライバシー保護のための仕様です。

ただし、**学年主任**は担当学年全体の統計を閲覧でき、**校長/教頭**は全学年の統計を閲覧できます。

---

## 3.2 トラブルシューティング

### トラブル 1: ログインできない

**症状**: メールアドレスとパスワードを入力してもログインできない

**対処法**:

1. メールアドレスとパスワードが正しいか確認してください
2. **メールアドレスは小文字のみ**です（大文字は使用できません）
3. パスワードは大文字・小文字を区別します
4. ブラウザのキャッシュをクリアして再度試してください
5. テストアカウント一覧は [1.4 テストアカウント一覧](#14-テストアカウント一覧) を参照してください

---

### トラブル 2: 画面が表示されない

**症状**: ログイン後、画面が真っ白、またはエラーメッセージが表示される

**対処法**:

1. ブラウザのキャッシュをクリアしてください
2. 推奨ブラウザ（Chrome,）を使用してください
3. ブラウザの JavaScript が有効になっているか確認してください
4. ブラウザのコンソールにエラーが表示されていないか確認してください

---

### トラブル 3: 連絡帳が作成できない

**症状**: 「提出」ボタンをクリックしてもエラーになる

**対処法**:

1. **1 日 1 件まで**の制約があります（意図的な制約）
2. 既に今日の連絡帳を作成済みの場合は編集してください
3. 全ての必須項目（体調、メンタル、振り返り）を入力しているか確認してください
4. 記載日が前登校日であることを確認してください

---

### トラブル 4: 担任ダッシュボードにデータが表示されない

**症状**: 担任ダッシュボードに生徒の連絡帳が表示されない

**対処法**:

1. 担当クラスの生徒が連絡帳を提出していない可能性があります
2. 生徒アカウントでログインして連絡帳を作成してください
3. クラスの担任として正しく設定されているか確認してください（管理者に問い合わせ）
